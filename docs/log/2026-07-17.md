  - `BUG-11` (CRITICAL): Identified a simulator vs. synthesis mismatch where the shift operator `<<` dynamically expands widths during simulation instead of preserving the LHS width (violating the checker semantics in `checker/widths/ops.rs`). Logged to `docs/audit/bugs.md`.

## Full CTO review report (second session)

- **Report**: Wrote the full review to `docs/audit/review-2026-07-17.md` (ratings,
  10-area findings, top-10 priorities, roadmap); indexed it in the audit README's
  audit-log table. Test suite verified at HEAD: 861/861 (31 suites).
- **BUG-11 escalated with proof**: end-to-end repro — `y = (a << 2) >> 2` on
  `bits[8]`, `a = 255`: `mimz eval` returns 255, Icarus on the emitted Verilog
  returns 63, and the program passes `mimz check` clean. Confirms the sim's
  width-growth (`value.rs:709`) vs the checker's width-preservation
  (`ops.rs:487`) is user-visible sim-vs-hardware divergence.
- **BUG-12 corrected**: the checker rejects a module-const reference inside a
  `fn` body too (E0101, reproduced at HEAD) — checker and emitter agree, so this
  is a consistent file-scoping design limitation, not broken emitter scoping.
  Recommended re-filing as a MEDIUM language-design item plus a spec ruling on
  `fn` scope.
- **Root cause named**: three independent semantic authorities (checker widths,
  sim `Val`, emitter width strings) + checker-skipping `eval`/`sim`/`test`
  paths. Recommended: check-before-sim, shared width/const-eval module, IR as
  correctness infrastructure; Verismith-style sim-vs-Icarus differential fuzzing
  as the structural regression defense.
