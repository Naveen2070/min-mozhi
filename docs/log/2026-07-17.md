## `?` valid-bundle sugar and `??` operator (Tasks 6-12)

- **Shipped**: `bit?`/`bits[N]?`/`signed[N]?` desugar at parse time to a
  compiler-synthesized `{ valid: bit, data: T }` bundle (`__Valid`/
  `__ValidSigned`), reusing feature 2.9's structural bundle machinery
  instead of a new `Type` variant; construction is bundle-literal only. New
  `??` operator (lowest precedence, left-assoc, chains): unwrap
  (`T? ?? T -> T`) and OR-mux (`T? ?? T? -> T?`), decided by the right
  operand's shape, never coercing width (E0911/E0912). Both forms always
  lower to an ordinary two-way mux in the emitter and the simulator — never
  tri-state. Diagnostics render `T?` back as surface syntax, not the
  underlying `__Valid(N: ...)` bundle name. Docs: spec/02 §1.12a, decision
  log, roadmap checkbox, CHANGELOG — see `docs/code/07-decisions-and-
evolution.md` and `spec/02-syntax-and-grammar.md`.
- **Review caught one Critical bug**: the OR-mux emission helper's first
  version (both crates) treated every `??` operand as a plain signal,
  which is wrong for a chained `x ?? y ?? z` — `y ?? z`'s result is itself
  bundle-typed, not a leaf signal, so extracting its fields the naive way
  produced syntactically invalid Verilog in the emitter (and would have
  been silently wrong in the simulator). Caught in Task 8's code review,
  fixed by recursing into nested `Coalesce` operands
  (`coalesce_field_expr`/`bundle_field_expr`) instead of rendering them as
  opaque signals; Task 10's simulator OR-mux was written with the fix
  already in place.
- **Two pre-existing `mimz-sim` gaps found and logged, unrelated to `??`
  itself**: `BUG-14` (FIXED, this feature's Task 9) — the simulator's
  elaborator never registered the `__Valid`/`__ValidSigned` builtin
  bundles in its own bundle registry, so any `?`-sugar-typed signal was
  totally broken in `mimz test`/`mimz eval` even though it checked and
  emitted cleanly. `BUG-15` (OPEN, scoped out) — `mimz-sim`'s
  `flatten_instance` and `fn`-call argument handling have no
  bundle-field-expansion baseline at all, unlike the emitter's four-site
  baseline; blocks OR-mux at those two sites in the simulator specifically,
  found while scoping Task 10. Both logged in `docs/audit/bugs.md`.
