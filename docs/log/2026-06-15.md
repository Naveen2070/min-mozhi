# 2026-06-15 — Robustness follow-up to the 2026-06-14 batch audit

A three-agent audit (plus manual code verification) of the 2026-06-14 batch
(built-ins, `fmt`, `--lang`/`morph`, LSP localization, `translate --order`,
security hardening, fuzz) found **no correctness bugs** and confirmed the
security pass (SEC-1..4) is sound and regression-tested. This entry records the
follow-up that closed the one real robustness issue, two low-severity edge
cases, and the cheap missing-test / doc-drift items the audit surfaced.

## Production fixes

- **Atomic `mimz fmt` write** (`src/main.rs` `fmt_file`). `fmt` overwrote its
  input in place with `std::fs::write`, so an interrupted / failing write could
  truncate the user's source. Now writes a sibling `<out>.<pid>.tmp` then
  `std::fs::rename`s over the target (cross-platform atomic replace; the PID
  suffix avoids concurrent-run collisions). The write still happens only after a
  successful lex + translate, so malformed input never reaches it.
- **`morph::inflect` empty-stem guard** (`src/morph.rs`). An empty `name` (e.g.
  a diagnostic whose span did not resolve, via `localized_msg`'s
  `src.get(..).unwrap_or("")`) produced a suffix-only fragment like `ஐ …`. Now
  returns empty for an empty stem.

## Tests (+11 `cargo test`: 247 → 259, of which 175 lib unit)

- **morph** (2 lib unit): `majority_breaks_ties_toward_the_earliest_keyword_column`
  (locks the "earliest tied column wins" rule, en < tl < ta, using the three
  flavors of the same `module` keyword for exact counts) and
  `inflect_of_an_empty_stem_is_empty_not_a_bare_suffix`.
- **checker** (5 lib unit): two-literal `min(5,10)` → E0407; `nand` of a bare
  `bit`; nested `abs(min(a,b))` and `min(abs(x),abs(y))` type composition; `abs`
  at the width boundary (`signed[127]` → `signed[128]`). Plus a CLI fixture
  `e0407_min_two_literals.mimz` (~72-file corpus).
- **parser** (1 lib unit): `a_long_flat_binary_chain_parses_without_tripping_the_depth_guard`
  — a 5000-term `a + a + …` chain parses cleanly, locking in that chain LENGTH
  is iterative and distinct from nesting DEPTH (the audit's "unguarded
  `binary()`" flag was a false positive — the precedence climb recurses only by
  a constant number of levels and every operand routes through the guarded
  `unary()`).
- **emitter** (1 lib unit): a built-in lowers fully-parenthesized inside a
  larger expression (`a & min(b,c)` → `… ((b < c) ? (b) : (c))`).
- **fmt** (2 integration): `-o` onto the input path round-trips via the atomic
  write (and leaves no stray `.tmp`); an unknown `--to` is a clean non-zero
  error that does not clobber the file.

## Fuzzing

- New `fuzz/fuzz_targets/pretty_roundtrip.rs` (third cargo-fuzz target):
  `lex → parse → pretty_print → re-lex → re-parse` must succeed, and an
  emittable program must round-trip to byte-identical Verilog. Wired into the
  per-PR `fuzz` job (60 s) and the weekly `fuzz-nightly` job (10 min), with the
  example corpus seed. CI-verified only — libFuzzer needs nightly/Linux and does
  not build on the Windows dev box.
