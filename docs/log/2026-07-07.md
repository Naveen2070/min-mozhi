## Issue Audit: UART and PID Implementations

User reported four issues encountered while working on UART echo and PID logic. Conducted an audit and categorized them:

1. **UART echo bit-indexed reg assign (`shift[bit_idx] <- rx`)**: Logged as **BUG-8**. The AST/grammar supports this, but the simulator explicitly errors out as unimplemented. Workaround is to use standard bitwise shift/masking.
2. **UART echo timing (+1 tick for start bit)**: Verified as expected hardware behavior (latency to detect the falling edge), not a compiler issue.
3. **PID saturation wrong**: Logged as **BUG-7**. The simulator's `eval_fn_call` improperly masks negative arguments without sign-extending them, causing e.g. `-128` to evaluate as `+128` in wider contexts. Workaround is using inline built-ins (`min`/`max` or equivalent) which correctly handle sign-extension.
4. **VGA bundle field access (`vga.hsync` rejected)**: Confirmed as a known, deferred feature per `spec/02-syntax-and-grammar.md` (v0.2.18). The workaround (using the flattened `vga_hsync` name) perfectly mirrors what the Verilog emitter outputs.

`docs/audit/bugs.md` has been updated with BUG-7 and BUG-8.

## Hardware Emulation Design Session

Conducted an interactive design session to plan physical hardware emulation for the built-in simulator.
Key decisions captured in \docs/Ideas/hardware_emulation.md\:

1. **Explicit Binding:** Hardware emulation will use explicit \sim { ... }\ blocks inside \ est\ blocks rather than implicit port-name matching.
2. **Syntax:** \Bind\ for connecting ports to peripherals, and a function-like \speed mhz(50)\ for defining real-time execution bounds (avoiding complex lexer changes for unit suffixes).
3. **Backend:** Will utilize secure, vetted Rust crates (\cpal\, \crossterm\).
4. **WASM Fallback:** Emulation will be parsed but act as a no-op in WASM during Phase 1 to focus on a stable native CLI experience.
