# 2026-07-09

## Hardware Emulation blocks added to Showcase Examples

Following up on the hardware emulation design session and the recently introduced `sim` and `test --emulate` capabilities, we integrated physical bindings into the existing showcase examples.

- **Speaker Emulation**: Updated the `melody_player.mimz` showcase to bind its `audio` output to the `speaker(waveform: "square")` peripheral and `playing` status to a `green` LED. This was applied across all five flavor variants (`english`, `tanglish`, `tamil`, `mixed`, and `tamil-pure/isai.mimz`).
- **UART Sockets**: Updated the `uart_echo.mimz` showcase to bind its `rx` and `tx` pins to `uart_rx(port: 8080)` and `uart_tx(port: 8081)` respectively. This creates a fully functional virtual serial port when emulating. Like the melody player, this was updated across all five flavors (including `tamil-pure/edhiroli.mimz`).

These changes leverage the native keyword translations (`paavnai`, `inai`, `vegam` for Tanglish; `பாவனை`, `இணை`, `வேகம்` for Tamil) as defined in the trilingual keywords table, strictly maintaining the four-flavor (and pure-tamil) equivalence rules.

## Simulator Pacing and Limits

- **Speaker Pacing**: Rewrote the `speaker` peripheral in `src/sim/emulate/speaker.rs` to synchronize the audio thread via a `mpsc::sync_channel(8192)`. This natively downsamples the simulated bits to the real audio sampling rate and perfectly paces the `harness.rs` batch simulation to real-time audio playback without sleeping artifacts.
- **MAX_SIM_CYCLES Bump**: Increased the hard-coded cycle limit from 1M to 500M to allow the `melody_player.mimz` 400M-cycle emulation to pass natively on a normal unthrottled run.
- **VCD Capture Limit**: Constrained `harness.capture()` to max 1M frames to prevent OOM on massive cycle counts.

## Emulation Bug Fixes: Speaker, Dashboard, and UART Timing

The MAX_SIM_CYCLES bump above just swapped a fast, clear cycle-limit error for a silent multi-minute hang on plain `mimz test`, and the speaker/UART peripherals had real correctness bugs once actually exercised live. Fixed the underlying issues instead of just the symptoms:

- **Offline Audio Rendering**: The speaker's `mpsc::sync_channel` live pacing (previous entry) assumed the interpreter could out-run a declared 50MHz sim clock; measured throughput is ~1M cycles/sec even in release, ~50x short. Replaced it with offline rendering — `on_tick` just buffers downsampled bits, `finish()` plays the whole clip back in one shot once ticking is done. Also moved all cpal calls onto a dedicated thread; calling them from the sim thread (which also drives the dashboard) hung indefinitely.
- **Non-Live Sim-Block Skip**: `mimz test` without `--emulate` now skips a sim-block tick outright instead of running it unbatched at interpreter speed — the 400M-cycle emulated tick was taking many minutes.
- **Stale Peripheral Reads**: `notify_on_tick`/`notify_peripherals` now read live signal state via `Sim::peek` instead of the last _captured_ frame, which goes stale past the 1M-frame capture cap above.
- **`--step` Mode + Dashboard Dismiss**: Added `mimz test --step` for single-cycle stepping (Enter to advance, `q` to quit). The dashboard now waits for Enter/`q` at the end of any live test instead of closing immediately, and drains stale input on open (the Enter that launched `mimz` itself was instantly dismissing the screen).
- **UART `port` Config**: `uart_rx`/`uart_tx` accept a `port:` bind config for a fixed socket port, falling back to an OS-assigned one when omitted.
- **UART Start-Bit Timing**: Fixed `uart_rx` to hold the start bit for `cycles_per_bit + 1` cycles, not `cycles_per_bit` — a receiver's `Idle` state detects the start bit on a cycle that doesn't count toward its own bit timer, so the one-cycle shortfall corrupted every received byte by one bit position.
- **Showcase Cleanup**: Trimmed `melody_player`'s emulated test to a ~6M-cycle quick beep, added a self-checking `uart_rx` literal-source example, fixed `uart_echo`'s socket-bound `rx` conflicting with the same test's literal drive, and regenerated goldens for the pending `TICK`-as-parameter change.
