## `default` assignments + item-level `const if` Рђћ spec v0.2.17

### Decision: `default` keyword spelling

- **D-DEFAULT-1**: Promote `default` from reserved Рєњ active keyword. Tanglish: `iyalbu`, Tamil: `Я«ЄЯ«»Я«▓Я»ЇЯ«фЯ»Ђ` (PROVISIONAL, pending native review). Marking PROVISIONAL avoids blocking R11; any future native-confirmed spelling replaces the PROVISIONAL entry.

### Decision: `default` semantics

- **D-DEFAULT-2**: `default name <- expr` is legal ONLY inside `on` blocks. It is NOT a new expression form and NOT valid inside `if`/`else` arms within the `on` block body Рђћ only at the top level of an `on` block body.
- **D-DEFAULT-3**: The emitter MUST emit all `Default` nodes BEFORE other `SeqStmt`s. Verilog non-blocking assignments use "last scheduled wins"; emitting the default first means any conditional `<-` in the same always-block silently overrides it Рђћ exactly the intended semantics.
- **D-DEFAULT-4**: Each register may have AT MOST ONE `default` per `on` block (E0810). Multiple `on` blocks may each have their own `default` for the same register.
- **D-DEFAULT-5**: `default` target must be a `reg` (E0809). Wires are combinational and cannot have a default sequential assignment.

### Decision: `const if` scope

- **D-CONSTIF-1**: `const if` is MODULE-BODY only. It may appear anywhere a `ModuleItem` may appear, but NOT at file level (`TopItem`). File-level conditional items are out of scope for v0.2.
- **D-CONSTIF-2**: No new keyword. `const` and `if` are already active. The parser uses one-token lookahead: `const` followed by `if` Рєњ `const if` block; otherwise Рєњ `const` declaration. This avoids the Tanglish/Tamil review loop entirely.
- **D-CONSTIF-3**: The condition must be a compile-time constant evaluable by `consteval::eval()`. Runtime expressions produce E0811.
- **D-CONSTIF-4**: Items inside a `const if` branch follow all normal module-item rules. Ports, clocks, and resets MAY appear inside (no artificial restriction), but the checker validates the winning branch only Рђћ the losing branch is fully discarded after condition evaluation.
- **D-CONSTIF-5**: `const if` may be nested. The recursive nature of `ModuleItem::ConstIf` handles this automatically.
