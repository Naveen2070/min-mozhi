## Phase 1.5 kickoff — decisions logged (branch `phase-1.5-simulator`)

Started the Phase 1.5 (simulator) branch and recorded the kickoff decisions before

### Decision: build our own simulator + VCD writer (no external libraries)

- **Context:** Phase 1.5 needs an event-driven simulation kernel and a VCD
  waveform writer. The option of pulling a VCD/sim crate was weighed.
- **Decision:** hand-write both — the kernel AND the IEEE-1364 **2-state** VCD
  emitter. No external simulation or VCD dependency.
- **Why:** fits the project's minimal-dependency ethos (only serde/toml/clap
  today); full control over romanized signal names and module-matching `$scope`
  nesting in the VCD; smaller supply-chain / FLOSS-audit surface (the constitution
  mandates auditable FLOSS); easier WASM-playground builds later (idea 8.5); and
  it is consistent with the codebase already owning its lexer, parser, checker,
  emitter, diagnostics, and benchmark harness. VCD 2-state is a small ASCII format,
  and correctness is validated empirically by the Icarus differential suite +
  GTKWave opening the file, so a hand-written emitter is low-risk. Rejected: a VCD
  crate — it would only pay off for 4-state / real / time VCD, which Min-Mozhi
  does not use (2-state by design, mandatory resets).
- **Impact:** new `src/sim/` modules (kernel, elaboration, VCD, test runner) +
  `mimz sim` / `mimz test` commands, built in-house; no `Cargo.toml` dependency
  added for simulation.

### Decision: test-block execution syntax — support both `tick`/`expect` and `await` cycle-waiting

- **Context:** how `test` blocks are written/executed freezes at v0.1.0.
- **Decision:** support **both** the already-shipped `test "…" for M(args) { drive;
tick(clk[, n]); expect … }` (spec/02 §1.10) **and** an `await` cycle-waiting form
  where `await clk.cycles(n)` is exactly equivalent to `tick(clk, n)`. `expect`
  stays the assertion in both. **Method-await** (`let r = await uart.read_byte()`,
  idea 3.3) is **NOT in v0.1** — it needs callables/methods (currently E1110) and
  stays reserved.
- **Why:** `tick`/`expect` is already specced, trilingual (`kanam`/`uruthisei`),
  and teachable; the `await clk.cycles(n)` form is a thin additive timing sugar the
  founder wants for readability. Restricting `await` to cycle-waiting keeps the
  freeze surface small and avoids pulling in method-call infrastructure.
- **Open sub-decisions (resolve before building the await form, plan step B6):**
  (1) the **`await` Tanglish/Tamil spelling** — `await` is reserved English-only;
  activating it as a keyword needs all three columns, and the Tamil/Tanglish
  spelling must come from native review (never invented, R9/R11); (2) whether an
  `async` test-block marker is required (a new keyword `async`, not currently
  reserved) or `await` is usable in any `test` block (recommended: no marker);
  (3) `await <clock>.cycles(<expr>)` is a dedicated await-timing production that
  desugars to `tick`, NOT general method-call syntax.
- **Impact:** `src/parser/items/test.rs`, `keywords.toml` (`await` when the
  spelling lands), `spec/05-simulator.md`, the simulator's test runner.

### Decision: implement the test-header thamizh-order flip in Phase 1.5

- **Context:** the fifth grammar-engine word-order flip (`M() kaaga "…" sodhanai
{ }`) was deferred from Phase 1.8 because `test` blocks had no execution oracle.
- **Decision:** implement it in Phase 1.5 — execution is now the oracle. Completes

- **Impact:** `src/parser/items/test.rs` (thamizh-order test header), `spec/04`

### Decision: defer `sim::fatal` / `sim::warn`; 2-state kernel

- **`sim::` asserts** (simulation-only runtime diagnostics) are **deferred** to a
  later increment — `expect` covers test pass/fail for the first cut; `sim` stays a
  namespace prefix, not a keyword.
- **2-state only** (no X/Z) — reaffirmed (mandatory resets make 4-state

### Decision: keep the core 2-state; 4-state/real/time are not core (discussion)

- **Context:** asked why Min-Mozhi is 2-state (0/1) only — not 4-state (X/Z) like
  Verilog, nor `real`/`time` — and whether it could become so, whether the VCD
  writer would still work, and whether the emitter could be updated.
- **Why 2-state:** it is a principled identity choice, not immaturity. Mandatory
  resets mean every reg has a defined initial value, so there is no uninitialized
  state for `X` to model; 2-state avoids the X-pessimism/X-optimism sim-vs-synth
  mismatch class (hardware honesty); `Z` (tristate) is needed only at top-level
  pads (`inout`, reserved, Phase 2), never internally; and bits-only is simpler to
  teach.
- **Decision:** keep the **core language 2-state**. `real` (not synthesizable) is
  NOT a core type — the synthesizable answer to fractional math is `fixed[N,F]`
  (reserved, Phase 2); `real`/`time` belong only to a future verification/sim
  layer. 4-state, IF ever wanted, should be an **opt-in 4-state simulation/interop
  mode** (e.g. to mirror Icarus's X-propagation when debugging the differential),
  which is ADDITIVE — not a change to the core language guarantee. Treat full
  4-state in the core like `union`/`real`: a non-goal we do not reserve for.
- **Feasibility notes (for the record):** the VCD _format_ already supports
  `x`/`z` and `r<float>` natively, so it is never the blocker; our hand-written
  2-state VCD writer would need only a small extension to emit those value chars
  (owning the writer makes that easy). The emitter could likewise be extended —
  4-state would mean emitting `x`/`z` (the smaller part; the larger part is the
  type-system/checker work and dropping the no-unknowns guarantee), and `real`
  would only ever target sim/testbench output, never synthesizable RTL.
- **Freeze angle (R13):** making the _core_ 4-state after v0.1.0 would be breaking
  (it changes results of existing programs) and would need an Edition; a separate
  4-state _sim mode_ stays additive and can land anytime. Intent is firmly
  2-state. Captured in `docs/Ideas/simulator_ideas.md`.

## Phase 1.5 — B1 (elaboration) landed + console-trace decision

### Decision: opt-in console trace (`--trace`) on `mimz sim` and `mimz test`

- **Context:** the planned first cut gives `mimz test` pass/fail + teaching
  messages and `mimz sim` a VCD, but no `$monitor`-style cycle-by-cycle console
  signal table (the auto-printed log EDA Playground shows out of the box).
- **Decision:** add an opt-in console trace, **default OFF**, on both `mimz sim`
  and `mimz test`: `--trace` prints an every-cycle table; `--trace=changes` prints
  on-change (`$monitor`-style). Scope: default interface + state
  (inputs/outputs/regs); `--verbose` = all signals incl. wires; `--signals <a,b,…>`
  = an explicit subset (unknown names error cleanly).
- **Why:** the `$monitor` waterfall is the best "see it work" affordance for a
  beginner-first HDL (a learner without GTKWave gets instant terminal feedback),
  and it is nearly free — the kernel already produces a per-cycle value snapshot to
  feed the VCD, so the tracer is a second sink on the SAME snapshot (one source,
  two sinks ⇒ always consistent with the waveform). Additive CLI only — no language
  surface, no freeze impact.
- **Seam/scope:** lands at B5 (`sim`) / B6 (`test`); B2's kernel exposes the
  per-cycle snapshot both the VCD writer and the tracer consume. `sim::fatal` /
  `sim::warn` remain deferred (user-written log statements — a separate feature).
- **Impact:** `spec/05-simulator.md` (new §5 Console trace); the `mimz sim` /
  `mimz test` CLI; the kernel's snapshot seam. No new keyword.

## Phase 1.5 — B8 (differential vs Icarus + perf baseline) landed — Phase 1.5 feature-complete

Two independent validations of the event-driven kernel:

**Differential vs Icarus (Layer 3, `tests/icarus.rs::our_simulator_matches_icarus_bit_for_bit`).**
For each design, the test elaborates + runs it through OUR kernel in-process (the
exact `mimz::sim::run` path behind `mimz sim`/`mimz test`), then generates a
Verilog testbench that applies the SAME default stimulus (reset the first cycle,
inputs held, clock toggled at the same period) to the emitted Verilog under
`iverilog`/`vvp`, and asserts the per-cycle OUTPUT values match **bit-for-bit**.
Covers the counter (register + sync reset + `+%` wraparound) and the shift
register (a held 1-bit input + `<< | extend` comb path). This is distinct from the
existing Layer 2 (Icarus vs hand-written semantic asserts): Layer 3 pits our
simulator directly against Icarus, closing the "no external tool needed, and it
agrees with one" exit criterion. The testbench is generated from the elaborated
`Design` (ports/widths/clock/reset), so it generalizes to more examples cheaply.

**Perf baseline (`tests/sim.rs::the_counter_kernel_clears_the_perf_baseline`).**
The bare `tick` hot path on the counter measures **~3.3M cycle-events/sec in
release** (debug ~0.58M) — the gate asserts ≥1M in release; debug only checks a
low sanity floor (it runs ~10× slower, so the 1M bar would be a false alarm).
Each `tick` is one clock cycle = several signal events, so cycles/sec is the
conservative lower bound on cycle-events/sec.

+1 Icarus differential + 1 sim integration (349 total). **Phase 1.5 (simulator) is
now feature-complete:** B1 elaborate · B2 kernel · B3 comb propagation · B4
stimulus · B5 VCD + trace + `mimz sim` · B6 `mimz test` · B7 test-header flip · B8
differential + perf. Still deferred within v1 (logged, additive): the `await
clk.cycles(n)` test sugar (needs the native-review spelling), `sim::fatal` /
`sim::warn`, and 4-state simulation.

Docs: spec/05 (§4 validation+perf, Status footer — engine fully built, DRAFT until
committed); `docs/code/10-test-map.md`. Gate: fmt + clippy `-D warnings` + rustdoc
`-D warnings` clean; full suite green (debug + release, `REQUIRE_IVERILOG=1`,
including the new Icarus Layer-3 differential). No commit yet (this batch is
committed by the founder). After this commit, the release step (CONTRIBUTING,

## Phase 1.5 — C1 (combinational sim + signed-aware differential) landed

The first full-parity follow-on increment (workflow: `docs/plan/full-parity-simulator-workflow.md`).

**Planned:** a clockless `mimz sim` path; a signedness-agnostic differential; and
broaden the bit-for-bit Icarus differential to all single-module examples.

**Done:**

- `comb_run` (`src/sim/run.rs`) — the kernel already settles a combinational DAG
  (`Sim::snapshot()` needs no clock); `comb_run` captures one settled frame per
  input vector. `mimz sim` now routes a clockless design here instead of erroring:
  `--in` = one frame, new `--sweep a=0|1|2,b=3|4` = the cartesian product, one frame
  each. Same VCD/console-trace path. (`parse_sweep`/`sweep_vectors` in
  `commands/helpers.rs`; `--sweep` flag in `main.rs`.)
- Differential refactor (`tests/icarus.rs`): a single auto-routing `differential()`
  (clocked → default stimulus; combinational → generated input vectors), output
  values compared via Verilog **`%b`** (binary ⇒ signedness-agnostic, so `%0d`'s
  signed-print trap is gone), with per-example **param overrides** (`uut #(.LIMIT(3))`).
  Broadened from 3 → **12 ASCII-named english examples**: clocked (counter, shift
  register, edge detector, blinker@LIMIT=3) + combinational (adder, comparator,
  mux4, datapath, window, full_adder) + SIGNED (bitops, signed_math).
- **Bug caught by the new differential + fixed:** `signed_math` mismatched — our
  shared evaluator's lossless signed `+`/`*` (`src/sim/value.rs::binary`) used
  `l.bits.wrapping_add(r.bits)` (raw bits), so a negative signed operand was not
  sign-extended before the width grew (`+7 + (-2)` gave 21, not 5). `Sub` already
  used `as_i128`; fixed `Add`/`Mul` to match. This also corrects `mimz eval` for
  signed lossless math. Regression `signed_lossless_add_sign_extends`.

**NOT done in C1 (planned but carried forward):** the romanized **tamil-pure /
`vilakku`** examples are not in the bit-for-bit differential — their emitted Verilog
identifiers are romanized (differ from source names), so an auto-generated TB needs
the emitter's `romanize` applied on both sides. They keep Layer-1 + Layer-2. The
rest of full parity (instances C2, `repeat` C3, enum FSM C4) is out of C1 scope.

+5 lib unit (4 `comb_run` + 1 signed regression), +2 net sim integration; the

## Security/bug audit — bound the simulator's unbounded counts (SEC-5)

A full critical→medium audit (three parallel auditors, weighted to the new
Phase 1.5 simulator) found the **core pipeline clean** (SEC-1..4, BUG-1/2 intact;
all five thamizh flips depth-guarded; checked arithmetic throughout; no path
traversal) and **one real cluster**: the simulator skipped the "bound every
count" doctrine the rest of the repo follows. Fixes (SEC-5 / HARD-5 in
`docs/audit/`):

- **HIGH (untrusted input):** `tick(clk, n)` in a `test` block looped `0..n`
  (n from a test-block expression) pushing a frame each iter — `mimz test` on an
  untrusted `.mimz` could hang/OOM. Now bounded by `MAX_SIM_CYCLES = 1_000_000`
  (`src/sim/harness.rs`, fails fast).
- **MEDIUM (operator flags):** `--sweep` cartesian product had an unchecked
  `usize` multiply + no cap (`sweep_vectors` → `Result`, `checked_mul` +
  `MAX_SWEEP_VECTORS`); `--cycles` was an unbounded `u64` (clap range +
  `run()` guard).
- **LOW (defensive):** `translate.rs` `.expect("NameMap serializes")` → clean
  error; `mimz.toml` walk-up bounded (`MAX_CONFIG_WALK_DEPTH = 256`).

The auditors' multiply-overflow "highs" (`cycle * PERIOD`) are **unreachable**
once the loops are bounded (can't complete ~10^18 iters) — recorded as
checked-safe, no dead code added. New regression tests:
`a_tick_count_over_the_cycle_limit_errors_fast_not_hangs` (`tests/test_run.rs`),
`cycles_over_the_limit_is_rejected_by_the_cli` (`tests/sim.rs`),
`sweep_vectors_rejects_an_oversized_product` / `_allows_a_normal_product`
(`src/commands/helpers.rs`). +4 tests → suite **361**. Gate clean. No commit yet
(founder commits).

## Phase 1.5 — C2: module-instance flattening in the sim elaborator

The simulator now elaborates a module that **instantiates sub-modules**, including
across `import`s — closing the alu/chained gap. `src/sim/elaborate.rs`:

- New `elaborate_project(files, module, params)` builds a project-wide module
  registry (name → file+AST) and elaborates the entry module (`files[0]`);
  `elaborate(file, …)` now just wraps it with a one-file slice (back-compat).
- `ModuleItem::Inst` is **flattened** instead of rejected (`flatten_instance`):
  the child is elaborated under its bound params (explicit `arg` in the parent's
  consts, else the child default — mirroring `checker::widths::insts::child_binding`),
  then inlined with every child signal name-prefixed `{inst}_{name}`. Child inputs
  become parent wires driven by their connection expressions; outputs/wires become
  prefixed wires carrying the child's (rewritten) logic; clock/reset map to the
  connected parent signals; child regs/procs prefix-rename. The parent's
  `inst.port` field-read rewrites to the wire `inst_port` — exactly the
  `{instance}_{port}` name the Verilog emitter auto-declares (`emit_verilog/expr.rs`),
  so the flattened `Design` matches the emitted Verilog bit-for-bit. Instance→
  instance feeds work (chained's `fa0.cout` → `fa1.cin`). A small `map_expr`/
  `rewrite_seq` walker does the identifier/field substitution. `repeat` stays
  rejected (C3).
- `mimz sim` and `mimz test` now `load_project` (entry + imports) and call
  `elaborate_project`; `harness::run_test` takes the file slice.

Differential broadened to **18 examples** (12 english + 4 tamil-pure + **alu's
`Top`** instantiating the imported `Adder`, and **chained** chaining two
`FullAdder`s). +2 lib unit (`flattens_a_same_file_instance`,
`rejects_unknown_instance_module`; replaced the old `rejects_instances_for_now`),
no new integration `#[test]` (the differential covers the new examples), so the
suite is **361 → 362**. Verified `REQUIRE_IVERILOG=1` (iverilog 12): all 18 suites
pass, clippy `-D warnings` + fmt clean. Remaining sim parity: C3 (`repeat` →
ripple_adder) and C4 (enum FSM → traffic_light). No commit (founder).

## Phase 1.5 — C3 (`repeat` unroll) + C4 (enum signals): full simulator parity

The sim elaborator now handles the last two structural gaps; the elaborate-time
rewriting was unified into one `Rw` walker (enum-variant read → index literal,
`inst.port`/`arr[i].port` → flat wire, `match` variant pattern → index, plus the
loop-var / child-signal substitution map).

- **C3 — `repeat` unrolling** (`src/sim/elaborate.rs`): `ModuleItem::Repeat`
  folds `lo..hi` (capped at `REPEAT_BUDGET = 4096`, matching the emitter), and per
  iteration binds the loop var and inlines the body. Array instances `let fa[i]`
  flatten under the name `fa__{i}` (the emitter's `inst_name`), so `fa[i].port`
  reads resolve to `fa__{i}_port` and a chained `fa[i-1].cout` feeds the next
  stage. **Bit-indexed drives** (`sum[i] = …`) are collected per bit and assembled
  into a whole-signal Concat (widest bit first). Unblocks ripple_adder.
- **C4 — enum-typed signals**: a module's `enum` encodes each variant by index
  with width `clog2(variants)` — exactly the emitter's localparam encoding.
  `reg state: S` gets that width; `State.Red` reads fold to the index literal;
  `match` variant patterns rewrite to integer patterns (so the existing value
  evaluator needs no change). Unblocks the traffic_light FSM.

Differential broadened **18 → 21**: added ripple_adder (C3), traffic_light (C4),
and vilakku (Tamil-identifier toggler). **The differential's example list now
equals the emitter's entire single-file corpus — every example the compiler emits
also simulates bit-for-bit vs Icarus.** +2 lib unit
(`unrolls_repeat_with_instance_array_and_bit_drives`,
`elaborates_an_enum_signal_and_match`) → suite **362 → 364**. Verified
`REQUIRE_IVERILOG=1` (iverilog 12): all 18 suites pass, clippy `-D warnings` + fmt
clean. **Phase 1.5 full-parity simulator complete (C1–C4).** No commit (founder).
