- **No bug, overflow, security, or perf issue.** Catalog lookup is a `LazyLock`

  format-string risk); `localized_msg` slices the span with bounds-checked
  `src.get(..).unwrap_or("")` (UTF-8-safe for multibyte Tamil); a leftover `{`

## Close Phase 1.8 + a pre-freeze breaking-change audit

With Phase 0 and Phase 1 done and the v0.1.0 freeze near, reviewed the language

### Decision: close Phase 1.8 — `spec/04` DRAFT → stable

- **Context:** Phase 1.8's exit criteria 1 (same-AST/round-trip suites) and 2
  (panel sign-off, C3) were met; only criterion 3 ("`spec/04` bumped DRAFT →
  stable") was open. The deferred `test`-header word-order flip stays in Phase 1.5
  per the section 6 scope fence and does not block closure.
- **Decision:** bumped `spec/04` v0.2.4 DRAFT → **v0.2.5 stable** (R5: removing a
  DRAFT mark is a logged decision). Phase 1.8 marked 🟢 complete.
- **Why:** all three exit criteria are satisfied; the engine (four clause flips,
  lossless `--order`, 33/36 localized errors) is finalized and panel-ratified.
- **Impact:** `spec/04` header + footer; `docs/plan/phase-1.8-grammar-engine.md`
  (status + exit boxes); `docs/README.md` (spec-versions cell + phase row);
  `min-mozhi-roadmap.md` (phase heading). Docs only — no code, no test change.

### Decision: reserve `fn` + `function` before the freeze (the one real gap)

- **Context:** a three-agent audit of the spec/plan/idea docs found the language
  surface freeze-ready — v1 keyword set, type system, operators, `[hi:lo]` slices,
  monotonic chained comparison, the 8 safety rules, and E-code numbering are all
  locked, and 16 future features already carry reserved words. Every other planned
  feature is additive (tagged-union payloads, struct + spread `..`, async-reset,
  const-if, pipe `|>`, packages, `.method()` calls) or already reserved. The **one**
  planned feature needing a brand-new keyword that was **not** reserved:
  combinational **functions** (Phase 2 RTL parity; also unblocks pipe `|>`).
- **Decision:** reserve **both** `fn` and `function` now (R11), English-only;
  canonical spelling + Tamil chosen when the feature lands (likely `fn` canonical
  with `function` as an accepted alias, the `import`/`include` precedent).
- **Why:** R11/R13 — reserving pre-freeze keeps the later feature additive; doing it
  post-freeze would force an Edition. Reserving both spellings costs nothing and
  mirrors the `suzhal`/`சுழல்` dual reservation. Verification-layer type words
  (`string`, `union`, `real`/`float`) were considered and **deliberately not
  reserved** — they are non-goals for the synthesizable core; revisit if/when a
  verification layer is designed.
- **Impact (R11 full pipeline):** `keywords.toml` `reserved` list; `spec/03`
  reserved table + header bump v0.2.5 → **v0.2.6** + changelog; the VS Code grammar
  `invalid.illegal` rule; `src/lexer/keywords.rs` test `fn_and_function_are_reserved`
  (+1 lib unit, 302 → 303); `docs/README.md` keywords-version cell; `10-test-map.md`.
- **`..` non-issue:** `..` is already the range token (`0..8`) and can never be an
  identifier, so no reservation is needed; range-vs-spread is a later parser detail.

After this, **no known breaking change remains owed before the freeze.** The

### Decision: amend D7 — go public (and tag v0.1.0) after Phase 1.5, not 1.8

- **Context:** Phases 0, 1, and 1.8 are verified complete. D7 as written said the
  repo goes public after Phase 1.8 (now done), with the v0.1.0 tag (the R13 freeze
  point) at that gate. The founder asked whether to wait for the simulator
  (Phase 1.5) before going public.
- **Decision:** do **Phase 1.5 first**, then tag v0.1.0 + flip the repo public
  together. D7's public/tag gate moves from "after 1.8" to "after 1.5."
- **Why:** (1) a self-contained write → simulate → see-it-work loop (`mimz sim` /
  `mimz test` + waveforms, no external tool) is a much stronger debut for a
  beginner-first HDL, and "no external tool needed" is literally a Phase 1.5 exit
  criterion. (2) More importantly, tagging v0.1.0 freezes the language surface, and
  Phase 1.5 still settles a language-level decision — the `test`-block execution
  syntax (`tick`/`expect` vs `await clk.cycles(n)`, idea 3.3) and the deferred
  test-header word-order flip. Those are documented (spec/02 §1.10) but never yet
  executed; freezing before the simulator validates them would force any later
  change to ride an Edition + `mimz translate`. Doing 1.5 first keeps the
  break-freely-until-v0.1.0 doctrine (R13) honest. Rejected: tag/go-public now
  (freezes unexercised test syntax) and tag-now-but-stay-private (same freeze risk).
- **Impact:** `min-mozhi-roadmap.md`, `docs/plan/phase-1-verilog-backend.md`,
  `docs/plan/phase-1.8-grammar-engine.md`, and the two `phase-0-foundation.md`
