## Audit verdict — no production fix required

- **No bug, overflow, security, or perf issue.** Catalog lookup is a `LazyLock`
  `HashMap` (O(1), parsed once); `morph::fill` is literal `.replace` (no
  format-string risk); `localized_msg` slices the span with bounds-checked
  `src.get(..).unwrap_or("")` (UTF-8-safe for multibyte Tamil); a leftover `{`
  forces the English fallback. The wired arg keys (E0401 `{expected}/{found}`,
  E0402 `{op}/{lhs}/{rhs}`, E0408 `{first}/{second}`, E0601 `{type}`) all match
  their templates.
- The module split is behavior-preserving; `file()` correctly narrowed to
  `pub(in crate::parser)`; integration tests already exercise every split item
  parser. The two audit "MEDIUM"s were speculative (`show()` emits bounded
  `bits[N]`/`signed[N]` strings; superseded-exhaustiveness — now guarded below).

## Tests added (the real gap: localization coverage)

- **`tests/morph.rs` (+4 integration):** `e0402_interpolates_op_lhs_rhs`,
  `e0408_interpolates_first_and_second` (via a width-inferred operand position —
  a direct assignment raises E0401 first), `e0601_interpolates_type` — each
  asserts the localized Tamil renders the interpolated values with **no leftover
  `{token}`** (i.e. localization actually fires, not a vacuous English fallback).
- **`tests/morph.rs` (+1 guard) `message_catalog_placeholders_are_known_tokens`:**
  every `{token}` in an ACTIVE `messages.toml` template must be one `morph::fill`
  fills (the `{name}`/`{name.*}` set or a wired arg key). Catches a typo'd
  placeholder or an arg no diagnostic supplies — a _silent, permanent_ English
  fallback no other test would notice. (Scans template lines only; `#` comments —
  the doc header and the commented-out DEFERRED examples that mention
  `{reason}`/`{token}` — are skipped.)
- **`src/morph.rs` (+2 lib unit):** `arg_code_without_args_falls_back_to_english`
  (a `Diag` for E0401 with no args → `localized_msg` returns `None`, proving the
  leftover-`{` guard) and `fill_with_empty_name_leaves_no_stray_fragment`.
- **`tests/grammar_sync.rs` (+1 guard) `keywords_toml_has_no_superseded_spelling`:**
  hardens the #14 guard at the source — no superseded v1 spelling
  (`ulle`/`endral`/… and their Tamil forms) may reappear in `keywords.toml` as a
  canonical spelling or any alias. Lifted the shared `SUPERSEDED` list to module
