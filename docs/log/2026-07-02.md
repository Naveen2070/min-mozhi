# 2026-07-02

## Packages / namespacing lands — spec v0.2.19

The packages/namespacing feature went from spec to working code across every pass. Module/enum/bundle uniqueness is now per-file instead of project-wide, and `a.b.Name` qualified references disambiguate same-named declarations from different files.

- **Per-file uniqueness**: The checker now narrows E0001/E0002/E0909 from project-wide to per-file. Each import records its resolved file index so the checker knows which file a name came from.
- **Qualified reference resolution**: Core resolution logic landed — a dot-joined path like `a.b.Name` is matched against the file's visible imports. Two new error codes: E0110 when a bare name is ambiguous (visible from 2+ files), E0111 when a qualifier doesn't match any import.
- **Cache re-keying everywhere**: Internal memoization caches in `checker::scopes`, `drivers.rs`, `widths::Config`, and `clocks.rs` were all keyed by bare module name — they silently returned wrong data when two files declared same-named modules. Re-keyed to `(file, name)`. The `inst_output` check was also fixed to resolve independently of instantiation ordering.
- **Emitter and sim**: Both had their own parallel symbol tables (independent of the checker's) that needed the same multimap + qualified-resolve treatment. Emitter also got a fix for cross-file same-named modules producing colliding Verilog output.
- **Import matching**: Qualified paths now resolve against the referencing file's own import list, not a global table.
- **Hardening pass**: A post-implementation review found 4 gaps — a silent E0111 fallthrough, the widths/clocks cache bug (same class as the earlier fix, deferred), and the Verilog name collision. All fixed.
- **Test fixtures**: `e0110_ambiguous_reference`, `e0111_bad_qualifier`, and a `packages/` directory with `qual_{a,b,top}` exercising the full qualified-reference flow end-to-end.
- **Cleanup**: A deferred sweep gathered minor findings across Tasks 2–14 — comments, asserts, coverage gaps, small deduplications. Also fixed a broken private intra-doc link that was blocking the rustdoc R8 gate.

E-codes: 47 → 49 (E0110, E0111). Spec: v0.2.19, new §1.5b. No new keyword — reuses `import` paths as namespaces.
