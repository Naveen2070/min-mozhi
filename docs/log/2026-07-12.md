# 2026-07-12

## foreach shipped

Added `foreach <var> in <source> { ... }` as sugar over `repeat`/bare
`loop` — both an index-range source and an array/`mem`-element source,
usable at module-item level and inside `on`-block/`fn`-body statements.
New keyword (provisional Tanglish/Tamil spellings, pending native review),
new E0417 (element-form source must be array/mem-typed). Checker validates
the original `ForEach` node directly and delegates width/driver/clock
checks by lowering-and-recursing into the equivalent `Repeat`/`Loop` form;
emit/sim/pretty each lower on the spot too — no new codegen shape anywhere.
Closes the last open item in the loop family
(`docs/Ideas/language_plan.md` §10; `docs/plan/phase-2-suzhal-loop.local.md`).

### Decision: no SOV/thamizh word order for foreach

- **Context:** whether foreach should support Tamil SOV (subject-object-verb) word order.
- **Decision:** foreach stays code-order only, matching `repeat`/`loop`/`sync loop`.
- **Why:** those three constructs (which foreach lowers into) are not in
  `docs/guide/10-word-order-thamizh.md`'s six-construct flip list either;
  giving foreach alone a flip its own lowering targets lack would be inconsistent.
- **Impact:** SOV support for the whole loop family is a legitimate separate future item.
