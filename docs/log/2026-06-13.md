## Two pre-freeze syntax rulings landed (8.9 + 8.10)

The two ¬ß8 items the v0.1.0 freeze pressured are now settled and implemented.

### Decision: Allow the safe monotonic chained comparison (8.9)

- **Context:** `a < b < c` was a hard parse error (E1109). Idea 8.9 wanted
  Python-style `0 <= x <= 100`.
- **Decision:** allow a **monotonic one-direction** chain (all `<`/`<=` or all
  `>`/`>=`); it desugars to `&&` of adjacent pairs (`0 <= x < 100` ‚Üí
  `(0 <= x) && (x < 100)`), sharing the middle operand. Mixed-direction
  (`a < b > c`) and any chain with `==`/`!=` stay E1109.
- **Why:** the C trap (`(a<b)<c` silently wrong) was never legal here, so this
  is an **additive widening** ‚Äî no program breaks. In hardware the shared
  operand is a combinational value, so reading it twice is identical (no
  software eval-order subtlety). Reads mathematically, on-mission for
  education. Safe-by-construction (honesty/safety preserved).
- **Impact:** `src/parser/expr.rs` (`comparison_chain`); parser tests
  swapped (`chained_comparison_is_an_error` ‚Üí `monotonic‚Ä¶desugars` +
  `mixed_direction_chain_is_an_error` + `equality_cannot_be_chained`);
  spec/02 ‚Üí v0.2.6 (¬ß3 + table + precedence note); test-map; `language_plan.md`
  ¬ß9 marked done.

### Decision: Ratify `[hi:lo]` slicing as final, reject 8.10's break

- **Context:** idea 8.10 proposed replacing `data[3:0]` / `{a,b}` with
  Rust-style range slicing `data[8..16]` / spread concat ‚Äî a breaking change.
- **Decision:** **keep `[hi:lo]` and `{a,b}` as the canonical, final forms**;
  do not adopt range slicing (now or via an edition).
- **Why:** `[hi:lo]` (msb:lsb) is the universal hardware convention
  (Verilog/VHDL/every textbook); the spec keeps slicing/operators "universal"
  so a student stays fluent across tools. The breaking win is aesthetic and
  **fails the "benefits the future" bar** against that cross-tool cost. A second
  range form would also clash descending-inclusive vs ascending-exclusive ‚Äî a
  worse beginner trap than either alone.
- **Impact:** spec/02 ‚Üí v0.2.6 (¬ß1.8 note); no code change (slicing already
  works); `language_plan.md` ¬ß9 marked done. The `..` token is therefore free
  for the spread/splat family only (8.7/8.8), never ranges.
