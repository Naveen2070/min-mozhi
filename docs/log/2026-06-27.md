## `clog2` compile-time built-in (branch `feat/clog2-const-builtin`)

Exposed the ceil-log2 the compiler already computed internally (enum-signal
widths) as a user-facing **compile-time** built-in, alongside
`min`/`max`/`abs`/`nand`/`nor`/`xnor`. Spec: `spec/02` §1.8 (v0.2.13).

### Decision block (R13)

**What.** `clog2(n)` — the one _compile-time_ built-in (the rest are runtime
folders, `spec/02` §1.8). Folds to the bits needed to address `n` items
(`⌈log₂(n)⌉`, floored at 1). Valid wherever a constant is (a width
`bits[clog2(N)]`, a `const`, a `repeat` bound); a runtime value position is
E0407, a non-positive argument is E0202.

**Why floored at 1 (diverges from Verilog `$clog2(1) = 0`).** Min-Mozhi has no
zero-width signal — `bits[0]` does not exist — so `clog2` must floor at 1 for
`bits[clog2(N)]` to always be a legal width. This is the SAME function that
sizes enum signals (`clog2(variant count)`); the two now share **one** source
(`checker::consteval::clog2_bits`), and `emit_verilog`/`sim::elaborate`
delegate to it so an enum width and a `clog2` width can never drift.

**Why no new error codes.** Reused E0202 (constant-evaluation, for `n < 1`) and
E0407 (built-in misuse, for a runtime value position). No `ALL_CHECKER_CODES`
bump ⇒ no `explain.rs`/`messages.toml` i18n surface. `clog2` is a built-in name,
not a keyword, so the keyword set stays v1 (no native-speaker review, R11).

**v1 scope limit (deliberate).** The argument must be a literal or `const`. An
overridable module **parameter** stays a symbolic Verilog `parameter`, and
Verilog-2005 has no `clog2`, so `clog2(PARAM)` cannot be emitted — it is an
honest emit error, never a silently-wrong width (caught the hard way: the first
emit test showed `bits[clog2(DEPTH)]` lowering to `[(DEPTH)-1:0]`, dropping the
`clog2`). The parametric path (emit a Verilog-2005 `clog2` function +
localparam, matching the floor-at-1 semantics) is a planned follow-up — it is
what unblocks the `Fifo(WIDTH, DEPTH)` stdlib cleanup, so that refactor is
deferred with it.

### Follow-up the same day: parametric `clog2` (the deferred half)

Lifted the "parameter ⇒ emit error" limit. A `clog2(<parameter>)` in a **body**
width (`reg`/`wire`/`mem`) now lowers to a call of an injected Verilog-2005
`clog2` constant function (matching `clog2_bits`, floor-at-1), so the width
tracks an instantiation-time parameter override. The function is injected once
per module (`Emitter::clog2_fn_used` + a `String::insert_str` at the body top),
and only when a body width actually needs it (a folded literal `clog2` drags in
nothing). A `clog2(<parameter>)` in a **port** width is still an error
(`Emitter::emitting_port` guard) — the body-scoped function can't reach the
header port list.

**Icarus-validated** (the real bar): `iverilog -g2005 -tnull` lints the constant
function in a `reg` width clean, and a sim with `Ring #(.DEPTH(5))` counts
`1,2,3,4,0,…` — the override flows through the function. Gate: `fmt` /
`clippy -D warnings` clean / **486 tests** (was 476) green.

### stdlib `Fifo` `AW` → `DEPTH` (the downstream cleanup, done same day)

Refactored the FIFO to take `DEPTH` (entries) instead of `AW` (pointer bits):
`mem data: bits[WIDTH][DEPTH]`, `head`/`tail` are `bits[clog2(DEPTH)]`, `count`
is `bits[clog2(DEPTH) + 1]`. **API decision:** generalize the ring wrap rather
than require a power-of-two — `tail`/`head` advance `+% 1` but reset to 0 at
`DEPTH - 1`, so any `DEPTH >= 1` is a true DEPTH-deep queue (the old `+% 1`
natural wrap only worked at powers of two). Default capacity unchanged
(`DEPTH = 4` = old `2^AW`).

Propagated across all four keyword flavors (english edited, tanglish/tamil via
`mimz translate`, mixed hand-mixed) + the pure-Tamil twin `varisai.mimz`
(`சுட்டு` → `ஆழம்`, SOV wrap). All four flavors emit **byte-identical** Verilog
(verified by md5). Goldens regenerated (`std_fifo*`, `tamil_pure_varisai*`),
Icarus differential green, and a manual `DEPTH=5` (non-power-of-two) sim
confirmed correct ordering + overflow-ignore. Docs: `docs/guide/stdlib/fifo.md`
(also fixed a duplicated paragraph), `phase-4-ecosystem.md` gap marked resolved.
Gate: `fmt` / `clippy -D warnings` clean / **485 tests** green.

## Ponytail over-engineering audit — verified, two cuts applied

Ran a repo-wide over-engineering audit (5 parallel scans of src + non-src), then
checked every finding against the code before acting. Most did not survive
inspection — the large line-count estimates counted _relocatable_ lines (whole
files, passes, functions) as _deletable_, but merging files or passes moves code
without removing it. Applied only the two that were real, safe deletions:

- **`ms()` deduplicated** — the identical `fn ms(Instant) -> f64` in
  `commands/check.rs` and `commands/compile.rs` moved to `commands/helpers.rs`.
- **`REPEAT_BUDGET` centralized** — the emitter and simulator copies (which MUST
  agree by the differential-oracle invariant) now share one
  `pub const mimz::REPEAT_BUDGET`. The checker's copy was **left independent** on
  purpose: it is a graceful-degradation walk budget, a different concept that
  only coincidentally uses 4096.

### Decision: most audit findings rejected after code inspection

- **Context:** an automated over-engineering audit proposed 18 cuts (~800 lines).
- **Decision:** apply only items 14 (`REPEAT_BUDGET`) and 15 (`ms()`); reject the
  rest.
- **Why:** items 7 (`ident_expr`/`int_expr`) and 11 (`Resolver` trait) were false
  positives — the helpers are called 9x and the trait has two impls feeding one
  shared `eval`, so removing them adds code. Item 17 (`#[serde(default)]`) would
  be a bug (defaults 0, needs 1). Items 1/5/6 (parser-items collapse, symbols
  merge, expr_reads unify) were relocation not deletion, and 5/6 touch the R10
  E-code contract. The rest (2, 8, 9, 12, 13, 16, 18) were marginal or risk-heavy.
  Item 10 (nightly-bench) kept at the user's request. Conclusion: the codebase is
  already lean.
- **Impact:** `src/commands/{helpers,check,compile}.rs`, `src/lib.rs`,
  `src/emit_verilog/mod.rs`, `src/sim/elaborate.rs`. The full per-item verdicts
  are this log entry (the standalone audit report was discarded as transient). No
