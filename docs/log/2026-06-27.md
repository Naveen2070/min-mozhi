### Decision: keep docs/guide/11-toolchain.md order by workflow

- **Context:** the guide sections were in a mixed order; new commands needed placement.
- **Decision:** ordered sections by the natural user workflow: `init` → `check` → `compile` → `eval` → `sim` → `test` → `lint` → `repl` → `explain` → `translate` → `fmt` → `doctor` → `completions` → `eject` → `mimz.toml` → `--lang` → `lsp`.
- **Why:** lets a new user read from top to bottom; utility commands at the end.
- **Impact:** existing sections re-ordered. Sections before `mimz.toml` all describe compiler commands; sections after it describe configuration, diagnostics, and tooling features.
