### The feature in one paragraph

OR-patterns let a single match arm cover multiple variants of a tagged enum —
`Op.Add(a, b), Op.Sub(a, b) => a + b` — using `,` as the separator. The
compiler must verify that every alternative exposes the same binding interface
so the arm body sees a consistent set of names and types. **E0808** fires when
any alternative is missing a name present in another, or when the same name
maps to different widths across alternatives. A `_` wildcard contributes no
bindings, so `A(x), _ => x` is E0808 — the wildcard arm cannot satisfy `x`.

### 2. Bugfix: restore `cx.sigs` on arm exit + safety stubs

**Commit `f721969` (10:57).** Three fixes across checker, emitter, and sim:

- **`src/checker/widths/`** — `inject_arm_bindings` previously removed
  injected names from `cx.sigs` after checking each arm body (via
  `remove(name)`). This broke when a binding name shadowed an existing signal
  — the existing signal was silently deleted. Changed to return `(name,
prev_value)` pairs so callers can `insert(name, old_ty)` to restore the
  prior state. Affects both `check_expr` and `infer_ty` paths.

- **`src/emit_verilog/expr.rs`** — added `debug_assert!(field_w > 0, …)`
  before the zero-width skip, catching E0807 violations at debug-build
  runtime.

- **`src/sim/elaborate.rs`** — `unwrap_or_else` fallback for
  `inferred_total_width` now wraps body in `debug_assert!` that no variant
  has payload fields; same `debug_assert!` added for field width in
  `variant_bindings`.

- **`src/sim/value.rs`** — `Pattern::Variant` arm in `pattern_matches`
  changed from `Err("not supported yet")` to `unreachable!()` since the
  elaborator now lowers all variant patterns to `IntMask`.

### 3. Parser: reject empty `Variant()` parens + Write-arm sim test

**Commit `2bd8e51` (11:22).** Two changes:

- **`src/parser/items/file.rs`** — `A()` (empty parens after variant name)
  is now a parse error with code E1113: "empty payload list — use `A` not
  `A()` for a tag-only variant". The previous parser silently accepted
  `A()` and created a zero-field variant, which was ambiguous (D1).

- **`tests/sim.rs`** — added
  `sim_tagged_enum_write_arm_payload_extracted`: a `Packet.Write(addr, data)`
  pattern with a two-field variant proves both fields bind independently.
  `xor_out = addr ^ data` must be non-zero (if `addr` and `data` aliased,
  XOR would always be 0). Covers both Write and Read arms with known packed
  values.
