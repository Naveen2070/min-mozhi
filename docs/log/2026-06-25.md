Branch off `develop` (which carries `parse_recover` + the `Error` placeholder

### Architecture — pure lib analysis + thin LSP adapter

The symbol logic is synchronous and AST-only, so it lives in the **library**
(`src/analysis.rs`): tokio-free, unit-testable without tower-lsp, reusable by the
WASM playground later. `src/lsp.rs` (binary) stays the thin tower-lsp adapter —
the same lib/bin split as today's `analyze`/`to_lsp`. All analysis works in
**byte offsets**; only `lsp.rs` converts to/from the LSP UTF-16 `Position`.

### Done (6 tasks, subagent-driven, each TDD + per-task gate)

- **`KeywordTable::canonical_spellings(flavor)`** (`src/lexer/keywords.rs`) — a
  flavor's full keyword list (one spelling per `Kw`), the completion keyword
  source.
- **Symbol index** (`src/analysis.rs`) — `build_index(&[LoadedFile]) ->
SymbolIndex`: one pass collecting a `Symbol` (name, `SymKind`, `file_idx`,
  defining span, hover `render`, enclosing `module_idx`) per definition across
  every loaded file. `Error` nodes skipped everywhere (no cascade).
- **`resolve_at`** — smallest name span covering the cursor (definition or use),
  resolved by scope: enclosing module → same module any file → file-level →
  anywhere. Cross-file, in-tree. Works on `parse_recover` partial trees.
- **Test-block resolution** (scope extension, requested mid-branch) —
  `test "…" for M { … }` blocks resolve: the module-under-test name and the
  body's driven inputs / `expect` / `tick` signals scope to M's ports, even
  cross-file (a new `same_module_any_file` tier in `resolve_at` carries this).
- **`completions`** — in-scope identifiers (enclosing module's members +
  file-level consts/enums + every module name) plus keywords in the file's
  **majority flavor** (`morph::majority_flavor` + `canonical_spellings`); no
  cross-flavor keyword leak.
- **LSP wiring** (`src/lsp.rs`) — `offset` (the inverse of `position`), a
  document-text store (`Mutex<HashMap<Url, String>>`) updated on didOpen/
  didChange/didSave, hover/definition/completion capabilities, `load_for_features`
  (parse_recover + import walk, skipping `std.*` virtual imports), and the three
  handlers mapping the lib result to `Hover` / `Location` / `CompletionItem[]`.

### Key decisions

- **Parse path split:** features run through `parse_recover` (half-typed files
  work); **diagnostics stay on the strict `parser::parse`** — no checker cascade
  on broken input, the exact UX concern the recovery branch deferred.
- **Cross-file:** resolve to definitions in the current file AND on-disk imports
  (the existing import walk). Embedded `std.*` (virtual `std:` paths) degrade
  gracefully — hover renders, go-to-def returns nothing (no real file URI).
- **Completion flavor:** majority flavor only, no cross-flavor keyword mixing
  (respects the one-flavor-per-file norm; avoids inviting the W0001 lint).

### Deferred

hover render (English in v1); `ExprKind::Error` for expression-level recovery;
`did_close` cache eviction (the doc cache grows for the session).
