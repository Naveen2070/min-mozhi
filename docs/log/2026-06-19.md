## Interactive playground waveform ÔÇö stimulus controls + hover inspection

The waveform was read-only: `simulate` ran once with whatever flags were in the
command box. For a combinational adder with one fixed input that draws a flat
line ÔÇö there was no way to vary inputs without hand-typing `--sweep`, and no way
to read a value off the canvas. Maintainer asked for an interactive waveform.
Decision (after a one-question scope check): **live input controls + canvas
inspection**, with per-cycle editing for _clocked_ designs deferred (it needs a
core-simulator change + an R1 spec update).

What landed:

- **`src/runner.rs` ÔÇö `ports` command.** Elaborates the module and emits its
  interface as JSON: `{"module","clocked","inputs":[{name,width,signed}],
"outputs":[ÔÇŽ]}`. The browser builds input controls from this instead of
  re-parsing source. CLI surface untouched (runner-only ÔÇö it is the playground
  engine). `signal_json` renders each port; names are identifiers so no JSON
  escaping is needed.
- **`src/runner.rs` ÔÇö `sim --steps "a=3,b=5;a=7,b=1"`.** Explicit per-step input
  vectors (groups split on `;`, each parsed by the shared `parse_bindings`), fed
  straight into the existing `comb_run` ÔÇö distinct from `--sweep`'s cartesian
  product, so the two error if combined. Bounded by `MAX_SWEEP_VECTORS`. Rejected
  for clocked designs with a message pointing to `--in`/`--cycles`. New
  `parse_steps` helper beside `parse_sweep`.
- **`site/src/components/Playground.tsx` ÔÇö stimulus panel.** On source change
  (debounced) it calls `ports`, then renders an adaptive panel: an editable
  **step table** for combinational designs (one column per step, 0/1 toggles for
  1-bit ports, value fields for buses; add/remove steps) seeded with a short
  ramp so the wave moves on load; **held-input fields + a cycles control** for
  clocked designs. Any edit re-simulates (`sim ÔÇŽ --vcd`) and redraws live. The
  old command-box `simulate ÔľŞ` button was removed (the panel drives the wave; the
  console still runs arbitrary commands).
- **`site/src/components/WaveformViewer.tsx` ÔÇö hover cursor.** Pointer-move maps
  x Ôćĺ time index; the canvas redraws with a vertical cursor line and a value chip
  per signal (drawn on the canvas ÔÇö no DOM overlay, so no inline styles).

Verified: Rust gate green (`fmt`/`clippy -D warnings`/`cargo test` **417**, +4
runner units: `ports`├Ś2, `sim_steps`├Ś2), wasm32 rebuilt + web/node glue
regenerated, headless smoke test extended (asserts `ports` JSON shape and
`sim --steps`), `astro check` 0/0/0, `astro build` 23 pages. No new wasm binding ÔÇö
both ride the existing `runCommand`. Nothing committed (R12).
