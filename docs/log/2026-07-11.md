## Test-visibility gate bug found during doc sync

While updating `docs/code/10-test-map.md`'s test count, `cargo test-summary`
reported 241 passing ÔÇö down from the 663 recorded 2026-07-06. Root cause:
the 2026-07-10 workspace split set `default-members = ["."]` in root
`Cargo.toml` (intentional, for fast local iteration on the shell crate ÔÇö
`docs/plan/workspace-split.local.md`), but neither `.github/workflows/ci.yml`
nor `tools/test-summary`'s usage docs were updated to pass `--workspace`.
Plain `cargo test` / `cargo test-summary` / CI's `cargo test --all-targets`
had therefore silently stopped covering `mimz-core` (399 unit tests) and
`mimz-sim` (97 unit tests) ÔÇö 496 tests invisible to the gate for one day
(2026-07-10 Ôćĺ 2026-07-11). Same gap hit `cargo clippy --all-targets` and
`cargo doc --no-deps` in CI.

Fixed: added `--workspace` to the clippy/test/doc/build steps in
`.github/workflows/ci.yml`, and documented the requirement (with the
reasoning above) in `docs/code/10-test-map.md`. True total: **737 tests**
(`cargo test-summary --workspace`) ÔÇö the count was never actually wrong,
only invisible to the default invocation.

### Decision: `default-members = ["."]` stays, but the gate must always pass `--workspace`

- **Context:** the test-visibility gap above.
- **Decision:** keep `default-members = ["."]` for fast local `cargo
build`/`cargo test` on the shell crate (its own stated purpose per
  `workspace-split.local.md`), rather than switching default-members to the
  whole workspace. Instead, every gate that must be authoritative (CI,
  `docs/code/10-test-map.md`'s recorded count) passes `--workspace`
  explicitly.
- **Why:** switching default-members to the whole workspace would slow down
  the everyday shell-crate edit loop the split was designed to keep fast;
  the actual defect was an unupdated CI script, not the default-members
  choice itself.
- **Impact:** `.github/workflows/ci.yml` (clippy/test/doc/build steps),
  `docs/code/10-test-map.md`, `docs/BUILD.md` section 2.

### Decision: 3-crate workspace split (mimz-core / mimz-sim / shell)

- **Context:** the WASM playground build needed to be dependency-optional-
  free without feature-flag gymnastics, and the CLI's heavy deps
  (ratatui/crossterm/cpal, tokio/tower-lsp) had no business being anywhere
  near the pure compiler pipeline or the simulator. _(Backfilled
  2026-07-11 ÔÇö the original work happened 2026-07-09/10, commits
  `c3fefc3`, `6d7221b`, `9535106`, then `4e88aaf`..`ad75bf8`; no Decision
  block was written at the time.)_
- **Decision:** split along the pure/impure axis into three crates ÔÇö
  `mimz-core` (lexer/parser/ast/checker/emit_verilog/span/diag/etc, zero
  optional deps), `mimz-sim` (event-driven simulator + `runner.rs`, depends
  only on `mimz-core`), and the root shell crate (CLI, fs I/O, LSP,
  hw-emulation), which re-exports both under the old `mimz::` paths so no
  caller-visible import churns. Rejected the earlier 6-crate proposal
  (syntax/check/emit/sim/lsp/cli) as over-engineered ÔÇö no unique deps per
  stage, a blurry syntax/check border, and needless import churn for no
  payoff.
- **Why:** `mimz-core`/`mimz-sim` become wasm-clean by construction (no
  `default-features = false` needed anywhere downstream), `mimz-sim` loses
  every `hw-emulation` cfg branch, and the pure/impure boundary becomes
  compiler-enforced instead of convention-enforced.
  Full rationale: `docs/plan/workspace-split.local.md`.
- **Impact:** `Cargo.toml` (workspace members), `crates/mimz-core/`,
  `crates/mimz-sim/`, `src/lib.rs` (facade), `crates/mimz-wasm/`
  (repointed to depend on `mimz-sim` directly), `docs/architecture.md`,
  `docs/code/README.md`, `CLAUDE.md`'s project-structure tree, and ÔÇö per
  the gate-bug section above ÔÇö `.github/workflows/ci.yml`.

### Decision: `EmulationHost` trait ÔÇö invert the sim/emulate dependency

- **Context:** `sim/harness.rs` was welded directly to `sim/emulate/`
  (ratatui `Dashboard`, the peripheral registry), which blocked `mimz-sim`
  from being dependency-optional-free ÔÇö the whole point of the workspace
  split above. _(Backfilled 2026-07-11 ÔÇö original work 2026-07-09, commit
  `9535106`; not mentioned in that day's log prose at all.)_
- **Decision:** define one narrow trait, `EmulationHost` (`bind`,
  `direction_of`, `on_change`, `on_tick`, `drive`, `frame`, `finish`), in
  `mimz-sim`, with zero ratatui/crossterm/cpal types in its signature. The
  harness takes `Option<Box<dyn EmulationHost>>`; `None` keeps today's
  degrade path (no terminal / no `--emulate`) unchanged. The real
  implementation (`Dashboard` + peripheral registry + led/speaker/uart_*)
  moved wholesale to the shell crate as `mimz::emulate`, constructed only
  by `commands/test.rs` when `--emulate` and stdout is a tty.
- **Why:** dependency inversion was the only way to keep `mimz-sim` free of
  optional deps while keeping the emulation feature at all ÔÇö the
  alternative (feature-gating cfg branches throughout `sim/`) is exactly
  what the split was meant to eliminate.
- **Impact:** `crates/mimz-sim/src/sim/host.rs` (trait definition),
  `src/emulate/` (implementation, relocated from `src/sim/emulate/`),
  `src/commands/test.rs` (host construction + tty check).
