# 2026-07-04

## Git Hooks sharing and automation

- **Context:** Git hooks are stored in `.git/hooks/` locally and are not tracked by Git or shared when cloning. If contributors do not manually configure them, code validation checks can be bypassed.
- **Decision:**
  - Created a version-controlled `.githooks/` folder.
  - Moved local `pre-commit` and `pre-push` quality gate hook scripts into `.githooks/`.
  - Added a root-level `build.rs` script that automatically configures Git to use `.githooks` (`git config core.hooksPath .githooks`) whenever a developer compiles the project locally.
- **Impact:** Automatically ensures all developers use the predefined hooks without needing manual setup commands.

## Feature: Statement-based function bodies & early return (Part 2)

- **Context:** Continuation of the early return support implementation.
- **Implemented Changes:**
