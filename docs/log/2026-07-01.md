## Packages / namespacing — spec v0.2.19

### Decision: scope

- **D-PKG-1**: Function names (`fn`, `E0801`) are explicitly OUT of scope.
  The 4 covered reference points (module instantiation, test header, enum
  type, bundle type) live in dedicated non-expression grammar slots where
  `.` means nothing today. `ExprKind::FnCall.name` lives in general
  expression grammar, where `.` already means field access
  (`inst.output`, `bus.field`, `Enum.Variant`) — qualifying it risks
  colliding with real dot-chains and is a materially different, riskier
  grammar change. Revisit only if a real collision pain point appears.
- **D-PKG-2**: No `package` keyword. The namespace a reference qualifies
  into is exactly the `import` path the referencing FILE wrote, not a
  separate declared identity independent of file layout — reusing syntax
  that already exists beats a new reserved word for a "modest" backlog
  item (`docs/Ideas/language_plan.md` §10).
- **D-PKG-3**: Declaration-time uniqueness (`E0001`/`E0002`/`E0909`)
  narrows from project-wide to per-file. Ambiguity is detected at
  reference-time instead (`E0110`), which is the actual collision fix —
  narrowing declaration-time only (leaving resolution untouched) would
  have left the real pain (forced renames on names never actually used
  ambiguously) unsolved.
- **D-PKG-4**: `checker::scopes` and `drivers.rs::summaries` (internal
  per-pass memoization caches, previously keyed by bare module name,
  project-wide) re-key to `(file, name)`. Necessary for correctness once
  two same-named modules can both be legitimately instantiated in one
  project — a bare-name cache would silently return the wrong module's
  data for one of them.
- **D-PKG-5**: `emit_verilog::Project` and `sim/elaborate.rs`'s own
  parallel symbol tables (independent of the checker's) get the identical
  multimap + qualified-resolve fix, fixed narrowly at each site rather
  than extracting one shared cross-pass type — smaller diff, no refactor
  risk to already-working emit/sim code in the same change.
