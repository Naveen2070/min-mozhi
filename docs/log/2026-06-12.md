## Decision — language_plan.md feasibility tiers (2026-06-12)

Measured against the existing compiler (98 tests, checker passes 1–5),
prior art (Chisel, Bluespec, SpinalHDL, Filament, SecVerilog), and the
constitution (educational, polytechnic persona, Phase 1 due Dec 2026).

- **Tier 1 — already shipped**: properties/`private set` (= out-port
  scoping E0104/E0107/E0108), null-safety's goal (= E0501/E0502), the
  sound approximation of the "hardware borrow checker" (= single-driver
  rules), half of width inference (`CtInt` + lossless `+`).
- **Tier 2 — already planned**: exhaustiveness (next checker slice; new
  spec question — require a fallback arm even on fully-covered enums for
  bit-flip safety), clock-domain checking (= clock-ownership slice +
  Phase 2 multi-clock; `@ clk` and `sync.double_flop` are design inputs;
  best idea relative to effort), await-tests/`sim::`/step-back debugging
  (= Phase 1.5 simulator API).
- **Tier 3 — good, spec for Phase 2/3**: tagged unions with payloads
  (build FIRST after Phase 1; gives `Result` free), interfaces/bundles
  (then structural matching), channels tier-(a) Decoupled-style bundles
  only, `prove` blocks via emitted SVA + SymbiYosys (NOT in-house Z3 —
  same pattern as leaning on Icarus), wire type inference,
  `count_ones`-style builtins.
- **Tier 4 — rethought/rejected**: tri-state `bits[8]?` (not
  FPGA-synthesizable internally), `defer` (wires are continuous; default
  assignment expresses it), full auto-retiming (synthesis tool's job),
  module-returning closures (params already cover it), `pulse`/affine
  tokens (research-grade, blocked on channels), `system_fault` silicon
  side (sim side is Phase 1.5), `secret` taint (tractable à la
  SecVerilog but wrong audience — Phase 5+), `filter` (no runtime-sized
  hardware), full temporal borrow checking (SMT-grade reachability).
- **Cross-cutting costs**: every keyword ×3 flavors through
  keywords.toml + native review; every feature ×4 example folders;
  several idea samples contradict the current spec — each Tier 3 item
  needs a spec section before code.
- **First post-Phase-1 candidates**: tagged unions, interfaces/bundles,
  clock-domain checking.

Pointers added: phase-1.5 plan (simulator ideas), phase-2 plan (CDC
inputs), phase-4 plan (prove/SymbiYosys + parked items), roadmap bullet.

## Decision — constitution v0.3: modern-secure-HDL co-goal (2026-06-12)

The founder updated the language goal: Min-Mozhi is now ALSO a modern HDL
with compile-time security and modern syntax for better DX — **equal in
rank with the education goal** (founder's explicit choice over
"education still first" and "modern primary"). spec/01 bumped to v0.3:

- **Goals**: new co-primary goal #3 in section 1; second audience (modern
  developer wanting a secure, ergonomic HDL) added in section 2.
  Production completeness stays a non-goal.
- **New G5 — Secure by construction**: explicit-flow `secret` taint
  (SecVerilog lattice-label model; `secret`/`declassify` keywords; timing
  side channels explicitly out of scope) + fail-secure `system_fault`
  sticky-fault network v1 (FAULT_OUT pin, safe-state mux on outputs,
  lockout until cold reset; no clock gating).
- **Tie-breakers reordered** (founder's choice among three options):
  honesty > safety > **security** > readability/**DX** > speed >
  brevity > Tamil idiom. Security slots right after safety (both are
  correctness promises); DX merges into the readability rung.
- **Tier 4 re-triage under v0.3** (founder confirmed each):
  - `secret` taint **promoted** to Tier 3, slice 1 = explicit-flow only.
  - `system_fault` silicon side **promoted** — sticky-fault v1 spec'd
    for Phase 2/3; clock-stop stays parked.
  - `?` **re-targeted**: `bits[N]?` = valid-bundle (Chisel `Valid`
    style), `??` = mux on valid; blocked on interfaces/bundles.
    Tri-state meaning stays rejected (FPGA physics).
  - Salvage items added to Tier 3: `default` assignments (from `defer`),
    honest `pipeline(stages = N)` stage-inserter (from auto-retiming),
    item-level const-`if` (from module closures), must-consume lint on
    channel reads (from affine tokens).
  - Still rejected (reasons are physics, not priorities): tri-state,
    `defer` model, auto-retiming claim, module closures, full
    `pulse`/affine typing, `filter`, full temporal borrow check.
- New keyword budget created by this decision: `secret`, `declassify`,
  `default`, `pipeline` — each ×3 flavors through keywords.toml + native
  review before any code.
- **Second re-triage of Tier 4** (same day, founder asked whether any
  reject survives the security/DX lens): **1.1 borrow check moved to
  Tier 3 as "prove-backed shared-resource access"** — the user states
  the exclusion property in a `prove` block, SymbiYosys proves it, the
  checker then accepts the guarded double-drive (automatic version stays
  rejected — compiler-side reachability is SMT-grade). Tri-state row
  gained a note: **top-level `inout` pads** (I2C SDA, shared external
  buses) are a separate legitimate future feature, pad-level only.
  Everything else stays rejected on physics: `defer` model, full
  auto-retiming, module closures, full pulse/affine, `filter`.

## Decision — LSP v0 pulled into Phase 1, non-gating (2026-06-12)

Diagnostics-only `mimz lsp` (via `tower-lsp`: parse + check on open/save,
publish checker E-codes + help in-editor) moves from Phase 4 into Phase 1's
Visibility section. Rationale: (1) its prerequisites — lib/bin split +
`--json` diagnostics — are already Phase 1 items under the E-code retrofit;
(2) it IS the "second consumer" that architecture section 5 names as the
trigger for the lib split, making the split real instead of speculative;
(3) errors-as-you-type in the student's language is the teaching feature
(D4 visibility precedent — same reason syntax highlighting was pulled
forward). **Non-gating**: v0.1.0 and the safety slices outrank it.
Hover/go-to-definition/completion/`translate` integration stay in Phase 4.
