### Decision block (R3)

**What.** Combinational user-defined functions: `fn f(params) -> T { [let x = e …]
bodyExpr }`, called as `f(a, b)` in any expression. File-level only (not inside
modules or other functions). Pure combinational — no clocks, no registers, no
module instantiation.

**Why file-level only (D3).** Keeping functions at file scope mirrors Verilog's
`function automatic`, simplifies scope resolution (functions can't capture module
state — they see only their parameters and `let` locals), and avoids the question
of whether a function declared inside a module could reference that module's ports
(it cannot — the only inputs are explicit parameters). A function needed by two
modules sits at file level and the emitter injects it into each using module.

**Why monomorphic (D5).** Functions are not generic/parametric — a call `f(x)` must
resolve to exactly one `FuncDecl` at compile time. No instantiation, no type
parameters. This keeps the checker and emitter simple: one width-check per
declaration, one Verilog body per function.

**Why recursion is banned (E0805).** Combinational circuits are acyclic by
definition. A recursive `fn` call would create a combinational loop, which the
existing DAG checker (E0504) would catch at module level anyway — banning it at the
function level gives a clearer, earlier error (E0805 "recursive fn" is more
informative than E0504 "combinational cycle through a function").

**Why `let` locals are typed as `integer` in emitted Verilog.** `LocalLet` carries
no `ty` field (the type is inferred from the right-hand expression). Emitting
`integer` (32-bit signed Verilog) is a deliberate compromise: it covers most
intermediate values in practice and avoids a partial-width-inference pass. Width
inference for locals is a follow-up (requires an AST change to add a `ty` field
and a new checker sub-pass). Annotated with a `// ponytail:` comment in the
emitter.

**Why `function automatic` (not `function`).** Verilog-2005 `function` uses static
local variables — calling it twice with different args can alias. `function
automatic` uses stack-allocated locals per call, matching the expected purely
functional behavior. Static Verilog functions would be a silent semantic bug.

**Keyword.** `fn` (English), `function` (alias, familiar to web/systems developers),
`saarbu` / `சார்பு` (the Tamil mathematical term for "function") — PROVISIONAL
Tanglish/Tamil, pending native-speaker review (R9/R11).

**E-codes.** E0801 (duplicate fn name), E0802 (fn name shadows builtin),
E0803 (arity mismatch), E0804 (return-type width mismatch), E0805 (recursion);
E1110 reused for unknown fn name at call site.
All reuse the existing E0xxx/E1xxx namespace (no new code block needed).
