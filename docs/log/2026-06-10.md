## Decisions

### Decision: Compiler implemented in Rust

- **Context:** roadmap originally said "Python or Rust"
- **Decision:** Rust, from day one — handwritten lexer + recursive-descent parser
- **Why:** avoids a guaranteed rewrite when the simulator/synthesis phases need performance; single static binary = zero-friction install for students (goal G1); enums + exhaustive match fit compiler work; `miette`/`ariadne` give teaching-quality diagnostics. Rejected: Python (perf wall, env friction), Go (no sum types/pattern matching)
- **Impact:** roadmap Phase 0 decision record; architecture doc §3 crate layout

### Decision: Trilingual support = two layers over one AST

- **Context:** English/Tanglish/Tamil support is the project's identity; original roadmap never specified the mechanism
- **Decision:** Layer 1 (Phase 1): keyword skins — one grammar, three keyword columns in one lexer table, freely mixable per file, `minmo translate` lossless. Layer 2 (Phase 1.8): Grammar Engine — `thamizh-order` parser profile with SOV/postpositional clause forms, selected by a `syntax thamizh` directive, producing the identical AST
- **Why:** mixing keywords is the low-friction migration path (G1+G3); keyword-swap alone is not real Tamil (SOV), but full natural-language parsing is out of scope — flipping a small closed set of clause heads gets natural reading at recursive-descent cost
- **Impact:** spec/03 + spec/04 written; roadmap gained Phase 1.8; spec/01 non-goal reworded ("not in v0.1 core" → planned as Layer 2)

### Decision: Safety model ("safe like Rust", translated to hardware)

- **Context:** goal — Go/TS feel, Rust-grade safety, for an HDL
- **Decision:** seven compile-time rules (spec/02 §6): exact widths with lossless `+`/`-`/`*` growth and explicit wrapping `+%` family; single driver; exhaustive `match` / mandatory `else` on wire-driving `if`; mandatory reg reset values; `=` wires-only vs `<-` regs-only; clock/reset as dedicated types; combinational DAG check
- **Why:** each rule kills a classic Verilog footgun (truncation, latch inference, multi-drivers, X-state, blocking/non-blocking confusion, CDC, comb loops) at compile time with a teaching error
- **Impact:** spec/01 §G2 table, spec/02 §6, all examples written to demonstrate the rules

### Decision: Repo process rules adopted

- **Context:** need the repo to always tell the true story as plans evolve
- **Decision:** `docs/RULES.md` — sources of truth (R1), plan changes update plan+roadmap+log same day (R2), major decisions get spec bump + Decision log block (R3), append-only dated logs (R4), spec versioning (R5), conventions (R6), session-close checklist (R7)
- **Impact:** this log file is the first artifact of the process

### Decision: Language semantics batch (spec/02 v0.2)

- Rust-style precedence (bitwise > comparison; comparisons non-associative) — kills `x & 1 == 0`
- Logical ops: **both** `&&`/`||`/`!` (universal) and `and/or/not` (translated aliases) — documented G1-x exception; `mimz fmt --strict` normalizes
- `signed[N]` ships in v0.1 code: no mixing, `signed()`/`unsigned()` casts (replace orphan `signedval`), type-directed `extend`, signed-only negative literals & unary minus (lossless N+1)
- Added: `const` declarations (was a missing production), `repeat i: lo..hi` compile-time generation, `import` semantics (file-relative, project-unique names, no cycles), full test grammar (drives/`tick`/`expect`)
- Cut: `on fall` (reserved); division/modulo declared deliberately absent
- B-block accepted as recommended: no div/mod, multi-clock reg-ownership rule, reg-requires-reset, inout→Phase 2, memories→Phase 2, structs→post-P2, match ranges/wildcards→v0.3+, ASCII digits only, naming = linter warnings

### Decision: Trilingual policy (spec/03 v0.2)

- One canonical phonetic Tanglish spelling per keyword (no diacritics, no aliases) + did-you-mean; TN SCERT textbook alignment as selection criterion; panel = tech/coder friends, panel majority wins over founder once formed; new keywords repeat/tick/expect added, fall removed

- Solo dev; execution order **1 → 1.8 → 1.5** (grammar engine before simulator)
