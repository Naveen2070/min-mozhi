## `--emit-testbench` Param Resolution + Name Collision Fixes

- **Bug fix:** `test_env` for `--emit-testbench` only resolved a test's own
  explicit `(NAME: expr, …)` args, so a module parameter with a declared
  default that a test didn't re-pass was absent from `test_env`, breaking any
  width expression that referenced it. Now merges in `dut.params` defaults
  for anything not explicitly overridden, same order/semantics as
  `sim::elaborate::elaborate_module`. Args also now chain left-to-right
  (a later arg can reference an earlier one), matching `sim::harness::params`.
- **Bug fix:** Two tests with different free-text names could sanitize to the
  same Verilog module identifier (e.g. `"edge case"` and `"edge_case"` both →
  `edge_case_tb`), silently emitting two same-named modules into one file.
  Now tracked and rejected with a diagnostic naming both tests.

  new row in `docs/audit/README.md`'s audit log. 3 new regression tests in
