# 2026-06-17

## Security/bug/overflow audit of Phase 1.5 C1–C4 → SEC-6 hardening

A maintainer-requested security, bug, overflow, and issues analysis of the committed
C1–C4 work (combinational sim, instance flattening, `repeat` unrolling, enum
signals), the demos, and the synced docs.

### Verdict

Well-hardened overall. The earlier **SEC-5/HARD-5** runtime count caps
(`MAX_SIM_CYCLES`/`MAX_SWEEP_VECTORS`, clap range, `checked_mul` before alloc,
`saturating_add` tick guard), `REPEAT_BUDGET=4096`, the enum encoding (identical
to the emitter's `clog2` — differential parity), guarded shifts/masks, panic-free
enum/variant lookups, checked const-eval, and the demos all audited clean.

The gap: SEC-5 bounded the simulator's **runtime** counts but not its
**elaboration-time** ones, and `mimz sim`/`mimz test` run on the parsed AST
**without the checker** (which has its own recursion/cycle guard). Four findings,
all in `src/sim/elaborate.rs`, all fixed (SEC-6 / HARD-6):

### Findings + fixes (SEC-6)

- **SIM-1 (HIGH, untrusted-input DoS).** Instance flattening
  (`elaborate_module` ↔ `flatten_instance`) had **no depth/visited guard**. A
  recursive/cyclic instantiation (`module A { let u = A() … }`, A→B→A) overflowed
  the stack and **aborted** `mimz sim`/`mimz test` (reproduced: "has overflowed
  its stack"). Fix: `MAX_INSTANCE_DEPTH = 16`, threaded through both functions →
  clean error. Kept small on purpose — each level is a large frame and the bound
  must fire within the 1 MB Windows main-thread stack (a first cut at 64 still
  overflowed before tripping, exactly as the parser's `MAX_DEPTH` history warned).
- **SIM-2 (LOW, panic).** The `repeat` span `(hi - lo)` was a raw `i128`
  subtraction before the budget check — extreme bounds (`hi - lo` past
  `i128::MAX`) overflow-panicked. Fix: `checked_sub` → over-budget error. (A recon
  script wrongly cleared this assuming the subtraction was inside `const_eval`; it
  is not — verified by reading the code.)
- **SIM-3 (LOW, correctness).** A bit-index drive cast `idx as u32` after only a
  `< 0` check → silent truncation for `idx ≥ 2³²`. Fix: bound to `0..128` first.
- **SIM-4 (LOW, correctness).** Flatten merge used `BTreeMap::extend` — a parent
  signal named like a flattened `inst_port` wire silently overwrote. Fix: the
  merge errors on a name collision.

### Verification

`mimz sim` / `mimz test` on a self-instantiating module now exit non-zero with
"instance nesting exceeds 16 levels …" instead of crashing; the cpu/alu demos
(depth-1 instances) still run. +4 lib unit
(`recursive_instantiation_errors_not_overflows`, `extreme_repeat_bounds_error_not_overflow`,
`an_out_of_range_bit_index_errors`, `a_flatten_name_collision_errors` in
`src/sim/elaborate.rs`). Suite **368** (debug + release, `REQUIRE_IVERILOG=1`);
fmt + clippy `-D warnings` + rustdoc + prettier + markdownlint + docs_sync/grammar_sync
clean; `graphify` updated. Docs: SEC-6 in `docs/audit/security.md`, HARD-6 in
`docs/audit/hardening.md`, `docs/code/10-test-map.md`. No commit yet (R12 — maintainer
commits).

### Follow-up pass (same day) → SIM-5

A final docs + security re-audit (maintainer-requested) confirmed the four fixes
above are in place and the audit docs are internally consistent, and turned up a
fifth elaboration-time panic in the same untrusted-input class:

- **SIM-5 (LOW, panic).** `int_expr` — which substitutes each flattened child
  const as a literal — built a negative value as `Neg(int_expr(-v))`, a **raw
  `i128` negation**. `i128::MIN` (magnitude 2¹²⁷, one past `i128::MAX`)
  overflow-panicked. Reachable on the checker-skipping sim path: a child const can
  evaluate to `i128::MIN` via checked arithmetic (`(-i128::MAX) - 1`), and **every**
  child const passes through `int_expr` during flattening. Fix: `int_expr` is now
  non-recursive and takes the magnitude via `unsigned_abs` (always fits `u128`), so
  `i128::MIN` lowers to `-2¹²⁷`. The sim evaluator already used `wrapping_neg` /
  `as i128` (`src/sim/value.rs`), so only `int_expr` panicked.

Also fixed three stale **364** test-count references the docs audit found
(`README.md` badge + status line, `docs/plan/phase-1.5-simulator.md`); the audit
docs themselves (security/hardening/test-map) were already consistent.

+1 lib unit (`an_i128_min_const_elaborates_without_overflow` —
`src/sim/elaborate.rs`). Suite **369**; full gate re-run clean. SEC-6/HARD-6 in

## A1 — replication `{N{x}}` (pre-v0.1.0 RTL-parity batch)

First feature of the post–Phase 1.5 plan (branch `rtl-parity-release`,
`docs/plan/rtl-parity-release-workflow.local.md`). **Replication `{N{x}}`** —
the inner concatenation group repeated `N` times, `N` a compile-time constant;
lowers to Verilog `{N{...}}`. Purely **additive**, no new keyword (so it works in
all four flavors and is byte-identical across them), mirroring concatenation
through the whole pipeline.

- **AST** (`src/ast/expr.rs`): new `ExprKind::Replicate { count, parts }`.
- **Parser** (`src/parser/expr.rs`): the `{` primary now disambiguates — after
  the first part, a `{` means replication (the first part was the count),
  otherwise concatenation. No regression to `{a, b}`.
- **Checker** (`src/checker/widths/ops.rs` `replicate_ty`): width is
  `count * inner`, validated 1..=`MAX_WIDTH` (reuses **E0410**); a non-constant
  count is **E0201** (from `consteval`). No new E-code.
- **Emitter** (`src/emit_verilog/expr.rs`): `{N{...}}`.
- **Simulator** (`src/sim/value.rs`): evaluates the group once and repeats it,
  bounded by the 128-bit evaluator limit; `src/sim/elaborate.rs` rewriter
  threads it through flattening, so the kernel stays at emitter parity.
- Also: pretty-printer (`mimz translate`/`fmt` round-trip), the romanize walk,
  and the name/driver/clock traversals.

**Tests** (+7 lib unit): parser `replication_parses_to_replicate`,
`braces_without_an_inner_group_stay_concat`; checker
`replication_width_is_count_times_inner`, `replication_width_mismatch_is_e0401`,
`a_non_constant_replication_count_is_e0201`, `a_zero_replication_count_is_e0410`;
sim `replication_repeats_the_group`. New four-flavor example `replicate`
(`BASE_EXAMPLES` 17 → 18, golden `tests/golden/replicate.v`) and an entry in the
Icarus three-way differential (kernel == VCD == Icarus). No grammar change needed
(replication adds no keyword; `{`/`}` already highlighted). Spec `02` → v0.2.8.
Suite **369 → 376**. No commit yet (R12).

## A2 — don't-care `match` patterns `0b1??` (RTL-parity batch)

Second batch feature. A binary pattern may use `?` for a don't-care bit
(`0b1??`, the Verilog `casez` idiom). **Additive**, **no keyword**. Binary only
for this cut (hex don't-care is a later extension).

- **Lexer** (`src/lexer/{token,mod}.rs`): new `TokKind::MaskedInt { value, mask,
width, raw }`; the `number` scanner consumes `?` digits in a `0b…` literal and
  builds value/mask/width by hand (`from_str_radix` can't parse `?`), capped at
  128 bits (E1004 on a bad form). A plain `0b…` literal still lexes as `Int`.
- **AST/parser**: `Pattern::IntMask { value, mask, width, raw }`; `pattern()`
  accepts the `MaskedInt` token. Expression `primary` does NOT (a don't-care
  value is meaningless — natural parse error).
- **Checker** (`src/checker/widths/patterns.rs`): a masked pattern must match the
  scrutinee width **exactly** (E0409, reused) and earns **no** exhaustiveness
  credit — a `_` arm (or exact literal coverage) is still required (E0601).
  Masked-on-enum is E0409, like a number pattern. No new E-code. Overlap among
  masked arms is not diagnosed (first match wins; documented).
- **Emitter** (`src/emit_verilog/expr.rs`): lowers to `((s & 'bMASK) == 'bVALUE)`,
  both sized to the pattern width.
- **Simulator** (`src/sim/value.rs` `pattern_matches`): `(s & mask) == value`.
  (`elaborate.rs::pattern`'s catch-all clone and `names.rs` (Variant-only) needed
  no change.)
- **Grammar** (`mimz.tmLanguage.json`): binary-literal regex widened to `[01_?]+`;
  `grammar_sync` unaffected (it syncs keywords/reserved, not number regexes).

**Tests** (+6 lib unit): lexer `dont_care_binary_literal_lexes_to_masked_int`;
parser `dont_care_pattern_parses_to_intmask`; checker
`dont_care_pattern_must_match_the_scrutinee_width`,
`a_dont_care_match_still_needs_a_wildcard`, `a_dont_care_pattern_on_an_enum_is_e0409`;
sim `dont_care_match_picks_the_masked_arm`. New four-flavor `priority` example
(priority decoder; `BASE_EXAMPLES` 18 → 19, golden + the Icarus three-way
differential). Spec `02` → v0.2.9. Suite **376 → 382**. No commit yet (R12).

## A3 — falling-edge `on fall(clk)` (RTL-parity batch)

Third batch feature: the negedge sibling of `on rise(clk)`. Two design points
made this bigger than A1/A2 (maintainer-decided):

- **`fall` promoted from reserved to an active keyword** (KW_FALL). It had no
  native spelling, so the maintainer authorized **PROVISIONAL** Tanglish/Tamil
  placeholders for dev/testing — `irakkam` / `இறக்கம்` ("descent", the antonym of
  `yetram`/`ஏற்றம்` = "ascent") — clearly flagged in `lang/keywords.toml`, `spec/03`,
  and here, pending native review (a temporary, logged R9/R11 exception so the
  four-flavor tooling works before the v0.1.0 freeze). The R11 pipeline was run
  in reverse: out of the `reserved` list, into the keyword table + grammar
  keyword rule (out of the invalid-reserved rule) + spec/03; the lexer
  reserved-word test moved to `mem`.
- **Edge-aware kernel.** The cycle-based kernel was edge-agnostic. It now models
  a period as **rising edge → sample → falling edge** (`tick_edge`), so a
  `posedge` reg updates before a `negedge` reg within the period and the sample
  sits between them — exactly the differential testbench (clock high, sample,
  then clock low). Pure-`rise` designs are byte-identical to before (the fall phase is
  a no-op); the **`dual_edge`** example (a posedge reg feeding a negedge reg)
  now matches Icarus **bit-for-bit** (it failed first with the naive both-edges-
  per-tick model — caught the ordering bug). `Reg`/`Process` gained an `edge`;
  `run()` interleaves the snapshot between the two `tick_edge` calls; the harness
  keeps `tick` = a full period.

Pipeline: AST `OnBlock.edge` (`Edge::Rise|Fall`) + `Edge` enum; parser
`on (rise|fall)(clk)` both word orders; emitter `posedge`/`negedge`; sim as
above. **+4 lib unit** (parser `on_fall_parses_with_the_fall_edge`,
`thamizh_order_on_fall_parses_to_the_fall_edge`; emitter `on_fall_emits_negedge`;
kernel `dual_edge_negedge_reg_captures_posedge_within_a_period`); 2 lexer tests
renamed (`fall_is_an_active_keyword`, `a_reserved_word_is_an_error`). New
four-flavor `dual_edge` example (`BASE_EXAMPLES` 19 → 20, golden + the Icarus
three-way differential). Spec `02` → v0.2.10, `03` → v0.2.8, `04` note added.
Suite **382 → 386**. No commit yet (R12).

## A4 — memories `mem` (RAM / register arrays) (RTL-parity batch)

Fourth and largest batch feature: an **addressable memory**. `mem name:
<element>[DEPTH] = init` declares `DEPTH` cells of an element type
(`bit`/`bits[W]`/`signed[W]`); read combinationally (`m[addr]`, runtime address,
yields the element) and written on a clock (`m[addr] <- v` inside `on`). Lowers
to a Verilog packed-element `reg [W-1:0] m [0:DEPTH-1]` with an `initial` loop
seeding every cell — power-on init, so "no uninitialized state" holds without an
unsynthesizable whole-memory reset (the `reset` line clears registers only).
Maintainer decisions: **provisional Tanglish/Tamil** `ninaivagam` / `நினைவகம்` (the
established Tamil term for "memory", pairing with `reg`/`pathivedu`), flagged
pending native review (R9/R11); and **power-on init** over a per-cycle reset.

The read/write SITES needed no parser change — `m[addr]` reuses
`ExprKind::Index` and `m[addr] <- v` reuses `LValue` indexing. New work:

- **Keyword/lexer:** `mem` reserved → active (KW_MEM); `Kw::Mem`; REQUIRED_KEYS
  29 → 30; the lexer reserved-word test moved to `sync`.
- **AST/parser:** `ModuleItem::Mem { name, ty, depth, init }`; a `Kw::Mem`
  dispatch mirroring `reg` plus the `[DEPTH]` bracket and the mandatory `= init`.
- **Checker:** `Bind::Mem`; `Ty::Memory { width, signed, depth }` (inline, so
  `Ty` stays `Copy`); depth const-eval (E0410/E0201); init width vs the element
  (E0405/E0408); an indexed read/write yields the **element** type and
  range-checks the address against `depth` (E0406); `mem` in `repeat` is E0303; a
  whole-memory assignment is E0108; `=` on a memory is E0505; two writing `on`
  blocks are E0503. No new E-code.
- **Emitter:** `reg {w}m [0:(D)-1];` + an `integer __mimz_m_i; initial for (…)`
  seed; the indexed read/write already lowered via the existing `Index`/`lvalue`.
- **Simulator:** `Mem` in `elaborate` (+ flattening through child instances),
  bound to its writing block's clock/edge; a sparse `BTreeMap<(name,addr),Val>`
  cell store in the kernel (an absent cell reads init); the `Resolver` gained
  `is_mem`/`mem_read` so `value::eval`'s `Index` arm does a runtime memory read;
  `run_seq` commits indexed writes into `next_mems` (two-phase, OOR write
  dropped); memory holds across reset; cells stay out of VCD (internal-only).

**+9 lib unit** (parser `mem_declaration_parses_to_a_mem_item`,
`a_mem_without_an_init_value_is_e1104`; checker `register_file_passes`,
`a_non_constant_memory_depth_is_e0201`, `a_zero_memory_depth_is_e0410`,
`a_memory_init_that_overflows_the_element_is_e0405`,
`a_constant_address_past_the_depth_is_e0406`, `a_memory_inside_repeat_is_e0303`;
kernel `memory_write_then_read_round_trips_a_cell`) + the new
`mem_is_an_active_keyword` lexer test. New four-flavor `regfile` example
(`BASE_EXAMPLES` 20 → 21, golden + the Icarus three-way differential — held
stimulus writes and reads the same cell; the `initial` seed + nonblocking
indexed write match Icarus bit-for-bit). Grammar: `mem`/`ninaivagam`/`நினைவகம்`
added to the keyword rule, removed from reserved. Spec `02` → v0.2.11 (new
section 1.11 + `memDecl` grammar; also corrected two stale Deferred entries —
`on fall` and don't-care patterns — that A2/A3 had left), `03` → v0.2.9 (KW_MEM),
`04` note added. Suite **386 → 396** (261 lib unit). No commit yet (R12).

## A5 — asynchronous reset (`async reset`) (RTL-parity batch)

Fifth and final HDL feature of the batch. A reset declaration may be prefixed
`async` (`async reset rst`) to make it **asynchronous**: every always-block that
uses the reset is widened to `always @(posedge clk or posedge rst)`, so a
register clears the instant `rst` is asserted rather than on the next clock edge.
A plain `reset rst` stays **synchronous** (the default). **Active-high only** for
this cut — maintainer-decided to **defer active-low polarity** (no polarity keyword
is reserved yet, and it would need kernel + differential-testbench changes).

Maintainer decisions: **async-only scope**, and **provisional Tanglish/Tamil** for
`async` — `otthisaivatra` / `ஒத்திசைவற்ற` ("non-synchronous", the negation of
synchrony `ஒத்திசைவு`), flagged pending native review (R9/R11), so the feature is
four-flavor-clean.

The design is deliberately small and emitter-centric:

- **Keyword/lexer:** `async` reserved → active (KW_ASYNC); `Kw::Async`;
  REQUIRED_KEYS 30 → 31; the v0.3-backlog reserved test dropped `async`.
- **AST/parser:** `ModuleItem::Reset` became a struct variant
  `Reset { name, is_async }`; an `async` prefix before `reset` sets the flag.
  ~15 `Reset(_)` match sites updated to the struct form.
- **Emitter:** the sensitivity list gains `or posedge {rst}` when the design has
  an async reset; the `if (rst)` reset body is unchanged.
- **Checker/sim:** no behavioral change. The cycle-based kernel applies reset at
  the clock edge, and async vs sync reset are **observationally identical at the
  per-cycle sample points** (sub-cycle timing is out of the kernel's model), so
  `is_async` is an emitter-only distinction — `elaborate` just records the reset
  name. Documented in `elaborate.rs`.

**+5 lib unit** (lexer `async_is_an_active_keyword`; parser
`async_reset_parses_with_the_async_flag`, `a_plain_reset_is_synchronous`; emitter
`async_reset_widens_the_sensitivity_list`, `a_sync_reset_stays_clock_only`). New
four-flavor `async_reset` example (`BASE_EXAMPLES` 21 → 22, golden + the Icarus
three-way differential — the `or posedge rst` Verilog matches the kernel
bit-for-bit under clock-aligned reset). Grammar: `async`/`otthisaivatra`/`ஒத்திசைவற்ற`
added to the keyword rule, removed from reserved. Spec `02` → v0.2.12 (async note
in §1.2 + `resetDecl` grammar), `03` → v0.2.10 (KW_ASYNC), `04` note added. Suite
**396 → 401** (266 lib unit). No commit yet (R12).

## Workstream B — versioning + language edition

Completes Workstream A (A1–A5 all landed) and starts the release-prep
workstreams. The point is to separate the **two version axes** that were
conflated — the **compiler** (crate version) and the **language edition** — the
`rustc 1.x` vs Rust edition `2021` distinction.

- **New `src/version.rs`** — the single source for the language axis. `Edition {
variant, slug, year, code, date, summary }`; `EDITION_HISTORY` (a `const`
  table, oldest-first, surfaced in source); `current()` (the tail);
  `COMPILER_VERSION` (`env!("CARGO_PKG_VERSION")`); `KEYWORD_SET_VERSION`;
  `version_block()` (the uname-style render). The first edition is **Wingless
  Butterfly** (`wingless-butterfly-2026-1`; maintainer-picked codename, freeform /
  not R9-bound). `lang/keywords.toml`'s root `version` is now parsed and exposed
  (`KeywordTable::version`), and a test cross-checks it against
  `KEYWORD_SET_VERSION` and the edition `code` (all = 1).
- **`mimz --version`** prints the uname-style block (variant on top, then
  `(compiler)` / `(language)`), handled directly in `main.rs` so the variant is
  the true top line (clap's `long_version` would prepend the bin name); `-V`
  keeps clap's short crate version.
- **Verilog header** now carries both axes: `// Generated by mimz 0.1.0-dev
(edition wingless-butterfly-2026-1) — …` (the golden test strips the banner
  line, so no golden churn).
- **`CHANGELOG.md`** (new) — the human mirror of `EDITION_HISTORY`.
  **`spec/06-editions.md`** (new) — the design note: the two axes, the
  `variant-year-code` scheme, `EDITION_HISTORY` as source of truth, and
  `mimz translate` as the post-freeze migration path; the per-file `edition`
  pragma is deferred (R13).

**Crate version stays `0.1.0-dev`** — maintainer-decided to drop `-dev` only at the
actual `v0.1.0` tag (Workstream D), which keeps the compiler axis honest and
nicely demonstrates the two axes are independent (compiler still `-dev`, edition
already `wingless-butterfly-2026-1`).

**+3 lib unit** (`src/version.rs`: `current_is_the_last_history_row`,
`keyword_set_version_matches_keywords_toml`, `version_block_shows_both_axes`).
`version` registered in the lib crate-map (`src/lib.rs`) + `docs/code/README.md`.

## Workstream C — cross-platform release binaries + CI hardening

The release pipeline (prerequisite for the v0.1.0 tag and Phase 4's
npm/PyPI/WASM wrappers). **No source-code change** — workflows + docs only.

- **New `.github/workflows/release.yml`** — on a `v*` tag (+ `workflow_dispatch`),
  **two jobs**. `build` (matrix, **native runners**, no cross-compilation):
  `x86_64-unknown-linux-musl` (ubuntu, `musl-tools`), `x86_64-pc-windows-msvc`
  (windows), `x86_64-apple-darwin` (macos-13), `aarch64-apple-darwin`
  (macos-latest) — each builds `--release --bin mimz`, strips (unix),
  **smoke-tests `mimz --version`**, packages (`tar.gz`/`zip`) with the binary +
  README + LICENSEs + an `UNSIGNED.txt`, and uploads a run artifact (so
  `workflow_dispatch` builds are testable); `contents: read`. `release` (single
  job, **tags only**, `needs: build`, `contents: write`): downloads all archives,
  computes one uniform `SHA256SUMS` (on ubuntu, so Windows/Unix hashes never
  disagree), and creates the Release via `softprops/action-gh-release` — **title +
  body from `RELEASE_NOTES.md`** (line 1 = title, rest = body, HTML comments
  stripped), a new repo-root file **rewritten each release** (`CHANGELOG.md` stays
  the long history). Safe because the dep tree is **100% pure Rust** (no C deps /
  `build.rs`; iverilog is test-only), so each native runner yields a correct,
  self-contained binary. **Maintainer decisions:** Linux = **musl static**
  (any-distro); **ship UNSIGNED** for v0.1.0 (signing deferred — `UNSIGNED.txt` +
  install-doc note on the one-time macOS Gatekeeper / Windows SmartScreen allow).
- **CI hardening (`ci_plan.md` §3.1 + §3.4):** every third-party action in **both**
  workflows is now **SHA-pinned** to a 40-char commit SHA (resolved via the GitHub
  API; `# vX` comment) — `actions/checkout`, `actions/upload-artifact`,
  `actions/download-artifact`, `Swatinem/rust-cache`,
  `DavidAnson/markdownlint-cli2-action`, `dtolnay/rust-toolchain` (stable +
  nightly), `softprops/action-gh-release`.
  Least privilege: top-level `permissions: contents: read` in `ci.yml`, `write`
  scoped to `nightly-bench` + the release build job.
- **Install docs** (`docs/guide/01-getting-started.md`): a "download a prebuilt
  binary" path beside the source build, with the per-OS archive table,
  `SHA256SUMS` verification, and the **unsigned-binary** instructions.

**Verification:** both workflows are valid YAML (`yaml.safe_load`); locally the
**Windows-MSVC target builds and `mimz --version` smoke-passes** (the binary
prints the uname block). The musl + macOS targets validate on their native
runners. **The 4-target matrix is push-gated (R12)** — it runs only when the
maintainer pushes a `v*` tag or triggers `workflow_dispatch`; this work prepares the
pipeline, it does not push. No source change → suite stays **404**. No commit (R12).
