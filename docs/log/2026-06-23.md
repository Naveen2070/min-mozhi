### What was decided

- **FIFO uses an explicit `DEPTH` parameter, not `1 << AW`.** While building the
  FIFO I found the sim kernel (`src/sim/value.rs`) evaluates a parameterized
  `1 << AW` as 0, so `count != (1 << AW)` was always false — `mimz test` passed
  the empty/full assertions only trivially (`full = count == 0`). The emitter
  keeps `<<` symbolic so the _Verilog_ is correct, which is why Icarus would
  agree but our kernel would not. Rather than touch the toolchain (out of scope
  for this branch), the module takes `WIDTH`, `AW`, and `DEPTH` with a documented
  `DEPTH = 2^AW` contract — idiomatic since the language has no `clog2`. **This
  is a real sim-kernel const-eval bug worth a follow-up fix** (parameterized
  shift in a runtime expression).
- **Pure-Tamil names are semantic (a Tamil reader grasps the function from the
  name)**, following the established pattern (`kanakki`, `saalaivilakku`): queue
  → `வரிசை`, transmitter → `அனுப்பி`, dimmer → `மினுக்கி`, digit-display →
  `எண்காட்டி`. Pure-Tamil test _names_ must have distinct character lengths — the
  testbench emitter sanitizes a Tamil name to all-underscores, so equal-length
  names collide on the Verilog module name (hit on `varisai`/`anuppi`; reworded).
