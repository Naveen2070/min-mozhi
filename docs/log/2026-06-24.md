# Dev Log — 2026-06-24

## Decision: Native git hooks for local quality gates

- **Context:** CI-only gates meant fmt/clippy failures reached CI before the
  developer noticed. Manual `npx prettier --check` and `npx markdownlint-cli2`
  steps were easy to forget.
- **Decision:** Added native shell-based git hooks at `.git/hooks/`:
  - `pre-commit` — fast gates: `cargo fmt --all --check`, `cargo clippy --all-targets -- -D warnings`, `npx prettier --check "**/*.md"`, `npx markdownlint-cli2`.
  - `pre-push` — heavy gates: `cargo test --all-targets`, `RUSTDOCFLAGS="-D warnings" cargo doc --no-deps`.
- **Why:** Zero-dependency approach (no Node/package.json needed at the root).
  Shell scripts run in Git Bash on all platforms. Native hooks are the
  simplest thing that works.
- **Impact:** `.git/hooks/pre-commit` and `.git/hooks/pre-push` created. CI
  boundary in `.github/workflows/ci.yml` narrowed to `main` + `develop`.

## Done

- CI trigger narrowed to `main` + `develop` branches.
- Pre-commit hook: fmt, clippy, prettier check, markdownlint.
- Pre-push hook: tests, rustdoc (warnings-as-errors).
