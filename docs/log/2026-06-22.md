### Minor nits fixed (`src/commands/compile.rs`)

The review surfaced three low-severity rough edges in `mimz compile`, now fixed:

1. **Panic on a stem-less `--output`** (e.g. `--output ..`): the testbench path
   derived a name via `file_stem().unwrap()`. Now a clean error instead of a
   panic.
2. **Partial output on a testbench-emission error**: the main `.v` was written
   before the testbench was generated, so a testbench error left a stray `.v`.
   The testbench is now built BEFORE either file is written.
3. **Silent `--emit-testbench` with no `test` blocks**: now prints
   `note: --emit-testbench had no effect — no \`test\` blocks found in …`and
writes only the`.v`.

- Test: `tests/examples.rs::emit_testbench_without_test_blocks_notes_and_writes_only_v`
  (asserts success, the note, and that no `_tb.v` is written).
- Doc: `docs/guide/11-toolchain.md` `mimz compile` section notes the no-tests
  behaviour and the build-before-write ordering.
- Gates: `cargo fmt --check`, `cargo clippy --all-targets -D warnings`, and
  `cargo test` all clean.
