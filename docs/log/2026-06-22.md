## 2026-06-22: Fuzz crash fix — `?` guard in `reskin`/`push_guarded` + wasm_parity fix + site npm scripts

Fixed the `translate_roundtrip` fuzz crash where romanized output failed to re-lex.

**Root cause:** `is_word_byte` did not include `?`, so `push_guarded` did not insert a separating space when a `MaskedInt` token ending with `?` (e.g. `0b1?`) was immediately followed by a Tamil identifier. After romanization to ASCII, the output like `0b1?rrrram` was greedily consumed by the re-lexer's number loop as a single token, producing E1004.

**Fix:** added `|| b == b'?'` to `is_word_byte` in `src/translate.rs:143`. Two regression tests (`masked_int_q_does_not_glue_onto_romanized_identifier`, `masked_int_q_does_not_glue_onto_english_keyword`) cover both romanized name and keyword reskin paths.

Gate: `cargo fmt --check`, `cargo clippy --all-targets`, `cargo test` (290 lib + 130 integration) all clean; the pre-existing `wasm_parity` skip is unchanged.

`*.local.md` rule in `.gitignore`).

**What landed (parser side, the prerequisite):**

- **`Error(Span)` AST variant** on `TopItem`, `ModuleItem`, `SeqStmt`, `TestStmt`
  (`src/ast/mod.rs`) — a placeholder carrying the skipped span.
- **New entry point `parser::parse_recover(toks) -> (File, Vec<Diag>)`** that never
  discards the tree. `parse`'s strict `Result` contract is unchanged (refactored to
  share an internal `run`); the compile/sim/emit pipeline still refuses codegen on
  any error, so an `Error` node can ONLY originate from `parse_recover` and never
  reaches codegen. New `Parser::span_since(start)` sizes the placeholder.
- **Recovery sites** now push the matching `Error` node instead of dropping the
  broken construct: `items/file.rs` (file items), `items/module.rs` +
  `items/mod.rs` repeat body (module items), `items/seq.rs` (`on` body),
  `items/test.rs` (`test` body).
- **Exhaustive handling** of the new variants across every consumer (checker
  skips them silently — no cascade; emit/sim/pretty treat them as unreachable
  no-ops, documented, since the strict path guarantees they never arrive).
- **Tests** (`src/parser/tests.rs`, +4): good items survive around a bad one;
  top-level garbage keeps the following module; `on`/`test` blocks emit
  `Seq/TestStmt::Error`; strict `parse` still `Err`s. Suite 290 → 294 lib green.
