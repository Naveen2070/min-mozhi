### Minor nits fixed (`src/commands/compile.rs`)

The review surfaced three low-severity rough edges in `mimz compile`, now fixed:

1. **Panic on a stem-less `--output`** (e.g. `--output ..`): the testbench path
   derived a name via `file_stem().unwrap()`. Now a clean error instead of a
   panic.
2. **Partial output on a testbench-emission error**: the main `.v` was written
   before the testbench was generated, so a testbench error left a stray `.v`.
   The testbench is now built BEFORE either file is written.
3. **Silent `--emit-testbench` with no `test` blocks**: now prints
   `note: --emit-testbench had no effect ‚Äî no \`test\` blocks found in ‚Ä¶`and
writes only the`.v`.

- Test: `tests/examples.rs::emit_testbench_without_test_blocks_notes_and_writes_only_v`

### Decision: `extern` is the only architectural-idea item that gates the freeze

- **Context:** auditing `architectural_ideas.md` before the public release +
  v0.1.0 tag ‚Äî which of the five must happen _before_ the freeze vs after.
- **Decision:** reserve the `extern` keyword pre-v0.1.0; ship every other idea
  after the tag. Ideas 1, 2, 5 are additive/trigger-based; idea 4 is already
  enforced.
- **Why:** under the growth doctrine (2026-06-13) additive changes are
  edition-safe but a new keyword that was a legal identifier in v0.1 is breaking.
  Reserving the namespace now is the cheap, only-now-possible move (R11).
- **Impact:** the four plan files above; the keyword-reservation work itself
  (lang/keywords.toml + spec/03 + tmGrammar + lexer test) is queued as the top
  pre-release action, not yet executed (R12 ‚Äî awaiting maintainer go).
