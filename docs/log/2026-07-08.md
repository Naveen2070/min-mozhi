# 2026-07-08

## Added `sim` emulation blocks to examples

We wired up the new `led` hardware emulation peripheral (added in yesterday's PR) to several existing example and showcase files. This allows users to visualize their designs running with real-time paced LEDs when using `mimz test --emulate`.

- `examples/english/blinker.mimz`: Bound the `led` output to a red LED (`speed mhz(50)`).
- `examples/english/traffic_light.mimz`: Bound `red`, `yellow`, and `green` to their respective LED colors (`speed hz(10)`).
- `examples/english/vilakku.mimz` and `examples/tamil/vilakku.mimz`: Bound `ஒளி` to a yellow LED (`speed hz(2)`), fully utilizing Tamil keywords in the Tamil flavor.
- `demo/cpu.mimz`: Bound the `leds` bus to a red LED (`speed hz(2)`).

These changes demonstrate the new hardware emulation capabilities on the existing codebase without modifying the core functionality.

## `uart_tx` and `uart_rx` Socket Support for Emulation

Added TCP socket support to the UART emulation peripherals (`uart_tx` and `uart_rx`) in the simulator.

- `uart_tx` target now supports `socket` output, streaming decoded bytes to an accepted TCP connection.
- `uart_rx` source now supports `socket` input, feeding bytes from a TCP socket into the simulation.
- Fixed blocking behaviors to ensure the simulator's per-cycle tick loop doesn't hang: made `uart_tx` socket streams non-blocking, and ensured the socket-connect wait is a true one-shot timeout.
- Threaded socket-open failures correctly as `Result` instead of panicking, matching the codebase error style.
- Updated `spec/05-simulator.md` and `CHANGELOG.md` with the new capabilities.

## Speaker Hardware Emulation

Added a new `speaker` hardware emulation peripheral utilizing the `cpal` audio library to play bound bits as audible tones.

- Registered the peripheral with construct and bit-state logic.
- Ensured speaker silences on drop to avoid hanging tones leaking between sequential test blocks in a process.
- Updated CI workflows to install ALSA dev headers (`libasound2-dev`) required for the new hardware emulation support.
- Refactored `Peripheral::on_tick` to be fallible across the sim engine.

## Fuzzing and Emitter Fixes

- Fixed a bug where a zero-length array runtime index caused underflow in both the `sim` evaluator and `emit_verilog` backend when processing unchecked ASTs generated via fuzzing.
- Disabled default features for the `mimz` dependency in `fuzz/Cargo.toml` to optimize fuzzing performance.
