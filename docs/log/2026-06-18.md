# 2026-06-18

## Plan: pull the Phase 4 web playground forward to lead the v0.1.0 public launch

Founder asked whether to build a landing page + docs — and, if possible, a
playground to run mimz and see the waveform — before going public. Verified
feasibility against the code and locked a plan.

### Decisions (founder)

- **Build one static site — landing + docs + in-browser playground — _before_ the
  public flip** (launch with the differentiator, not a README alone). Founder-gated
  (R12); the public flip + `v0.1.0` tag remain Workstream D.
- **Custom Astro app** (not Starlight): content-collections render the existing
  `docs/guide/*` + `docs/spec/*` markdown (sourced, never duplicated) and islands
  host the playground.
- **Monorepo, not a separate web repo** — docs stay a single source of truth, the
  WASM playground can't version-drift from the compiler (built against the same
  `src/lib.rs`, byte-equal differential test in one suite), versioning is atomic. The
  Node toolchain is isolated under `site/`; `crates/mimz-wasm` stays **out of
  `default-members`** so the R8 gate never builds it for the host.
- **Waveform: a custom canvas/SVG renderer behind a swappable `WaveformViewer({vcd})`
  boundary**, with **Surfer** (surfer.dev) as the documented drop-in upgrade if VCD
  later gains x/z or large traces. The VCD string is the stable contract.
- **Deploy: Vercel on a founder-owned subdomain** (not GitHub Pages) — served at root
  so no `base`-path WASM/asset pain, first-class Astro, per-branch preview deploys for
  founder review. Build wrinkle: Vercel's image lacks Rust+`wasm-pack`; leaning toward
  building wasm+site in the SHA-pinned GitHub Actions and `vercel deploy --prebuilt`
  (reproducible, pinned toolchain), with Vercel-native rust-bootstrap as the simpler
  fallback. Decided at Step 6. **Note:** any deploy (even a preview) makes the _site_
  public while the _repo_ stays private — the "go public" gate still governs the repo
  flip + `v0.1.0` tag (R12).
- **Landing may be a little flashy** — a domain-themed animated hero (gates/waveforms),
  but as a lazy-loaded island with a `prefers-reduced-motion` static fallback and a
  light WebGL/SVG approach over heavy Three.js (audience: college labs). Deferred to
  Step 6 polish so it can't delay the functional site.

### Why it's low-risk

Exploration confirmed the compiler+simulator lib is **WASM-ready today** — pure-sync,
no C deps, no `build.rs`, nothing that breaks `wasm32`. The whole pipeline is already
string-in / string-out and in-memory (`lex → parse → check → transliterate →
Project::from_files → emit`; `elaborate_project → run/comb_run → to_vcd`). The only
net-new library code is a ~30-line `compile_string(source, imports)` wrapper that
skips the `std::fs` file-loading path. The WASM browser playground is already the #1
priority in [`phase-4-ecosystem.md`](../plan/phase-4-ecosystem.md), and its
prerequisites (lib/bin split, `--json` diagnostics) landed in Phase 1.
