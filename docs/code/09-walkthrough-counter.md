# 09 — Walkthrough: `counter.mimz`, Stage by Stage

One real example through the whole pipeline, with the **actual artifacts**
at every stage. Reproduce everything here yourself:

```text
cargo run -- check examples/counter.mimz --tokens   # stage 2 output
cargo run -- compile examples/counter.mimz -o counter.v   # stage 5 output
```

(Token offsets and the Verilog below were captured from the real
compiler. If this page ever disagrees with what those commands print,
the page is stale — fix it, per RULES R1.)

## Stage 0 — the source

`examples/counter.mimz`:

```mimz
module Counter(WIDTH: int = 8) {
  clock clk
  reset rst

  out count: bits[WIDTH]

  reg value: bits[WIDTH] = 0

  on rise(clk) {
    value <- value +% 1
  }

  count = value
}
```

(The file starts with three `//` comment lines, omitted here — which is
why the token offsets below start at byte 206, not 0.)

## Stage 1 — load (`project::read_source`)

The file is read and NFC-normalized. No imports here, so `load_project`
returns this single file. All spans below are byte offsets into this
normalized text.

Things to notice:

- The blank lines in the source produced **no extra Newline tokens** —
  runs collapse (`postprocess_newlines`).
- After `{` there is no Newline token either: `{` is a continuation
  token, so the newline following it is dropped.
- In the Tanglish version (`counter.tanglish.mimz`), the ONLY difference
  in this stream is the `flavor` field on the keyword tokens — same
  `Kw(...)` kinds, same everything else. That is the whole trilingual
  mechanism, visible in one place.

## Stage 3 — parse (`parser::parse`)

The AST, in abridged notation (real field/variant names; spans omitted):

```text
File
└─ items[0] = Module {
     name: "Counter",
     params: [ Param { name: "WIDTH", ty: Int, default: Some(Int 8) } ],
     items: [
       Clock("clk"),
       Reset("rst"),
       Port  { dir: Out, name: "count", ty: Bits(Ident "WIDTH") },
       Reg   { name: "value", ty: Bits(Ident "WIDTH"), reset: Int 0 },
       On(OnBlock {
         clock: "clk",
         body: [ Assign {
           lhs: LValue "value",
           rhs: Binary { op: AddWrap, lhs: Ident "value", rhs: Int 1 },
         } ],
       }),
       Drive { lhs: LValue "count", rhs: Ident "value" },
     ],
   }
```

Things to notice:

- `bits[WIDTH]` is `Bits(Ident "WIDTH")` — the width is an **expression**,
  not a number. Nothing has evaluated `WIDTH` (and at parse time nothing
  could — it's a parameter).
- `+%` became `BinOp::AddWrap` — a distinct operator in the AST, even
  though (stage 5) it currently emits the same `+` as lossless add. The
  distinction exists for the checker.
- No flavor anywhere. The Tanglish file parses to this **exact** tree.

## Stage 4 — project table (`Project::from_files`)

`modules = { "Counter" → &Module }`, `enums = {}`. With imports (see
`alu.mimz`) this is where cross-file module lookup gets its data.

## Stage 5 — emit (`emit_verilog`)

Real output, byte for byte:

```verilog
// Generated by mimz 0.1.0-dev — Min-Mozhi (மின்மொழி). Do not edit.

module Counter #(
    parameter WIDTH = 8
) (
    input wire clk,
    input wire rst,
    output wire [(WIDTH)-1:0] count
);
    reg [(WIDTH)-1:0] value;
    assign count = value;
    always @(posedge clk) begin
        if (rst) begin
            value <= 0;
        end else begin
            value <= (value + 1);
        end
    end
endmodule
```

Map each line back:

| Verilog                    | Came from                                                             |
| -------------------------- | --------------------------------------------------------------------- |
| `#(parameter WIDTH = 8)`   | `Param` with its default — passed through symbolically                |
| `input wire clk` / `rst`   | `Clock`/`Reset` items (declared order)                                |
| `[(WIDTH)-1:0]`            | `Bits(Ident "WIDTH")` — the symbolic width, `(expr)-1:0`              |
| `assign count = value;`    | the `Drive` item                                                      |
| `always @(posedge clk)`    | the `OnBlock` (rising-edge only)                                      |
| `if (rst) ... value <= 0;` | **generated** — `collect_assigned` found `value`, used its reset expr |
| `value <= (value + 1);`    | the `Assign`; `AddWrap` emits `+` (same-width Verilog `+` wraps)      |

Things to notice:

- The writer never wrote reset logic; the emitter generated the whole
  `if (rst)` branch from the `reg ... = 0` declaration.
- `(value + 1)` is parenthesized — the emitter parenthesizes every
  compound expression, no exceptions.
- Everything stayed symbolic: change the instantiation's `WIDTH` and the
  same Verilog works; no const-eval happened anywhere.

## Where the future checker slots in

Between stages 3 and 5. For this file it would: resolve `WIDTH`/`value`/
`count`/`clk`/`rst`, verify `value +% 1` keeps width `WIDTH`, verify
`count = value` widths match, confirm `value` has a single driver and
`clk` owns the `on` block. Same AST in, same AST (plus tables) out — the
emitter wouldn't change.
