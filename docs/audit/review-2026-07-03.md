## Security (4/4 still present)

### S1 [MEDIUM] `mimz sim` panics on bundle literal missing a field — 🔴 STILL PRESENT

- **Where:** `src/sim/elaborate.rs:1238–1244` (`bundle_field_expr`)
- **Why:** `sim`/`ports`/playground `run_command` go parse → elaborate **without** `checker::check`
  (`src/commands/sim.rs:94`, `src/runner.rs:299,385`), so the
  `unreachable!("checker should have rejected this")` fires on user input.
- **Repro:** `out rsp: Handshake` driven by `{ valid: a }` → panic. (`mimz check` correctly reports E0901.)
- **Fix:** make `bundle_field_expr` return `Result<Expr, String>`; both callers
  (`elaborate.rs:561,624`) already propagate `?`.

### S2 [MEDIUM] `mimz eval` panics on enum-variant match pattern — 🔴 STILL PRESENT

- **Where:** `src/sim/value.rs:476` (`pattern_matches`, `Pattern::Variant` arm)
- **Why:** the eval path (`commands/eval.rs` → `sim::comb::eval_outputs`) evaluates the raw AST
  without `sim::elaborate`, the only place variants are lowered — so a raw variant pattern
  reaches the `unreachable!()`.
- **Repro:** `y = match a { State.Red => 1, _ => 0 }` with no enum declared → panic.
- **Fix:** `pattern_matches` already returns `Result<bool, String>` — replace `unreachable!` with `Err(...)`.

### S3 Root cause for the panic class — 🔴 STILL PRESENT

Commits ad08bdd, c26704a, 874769d fixed this same class one stub at a time.
**Running `checker::check` before `elaborate` in the sim/eval command paths closes the whole
class permanently** — otherwise expect more whack-a-mole from the fuzzer.

### S4 [LOW] Fuzz coverage gap: simulator path not fuzzed — 🔴 STILL PRESENT

## Performance (7/7 still present)

The simulator's per-event cost is dominated by String allocation and map cloning, not arithmetic.

### P1 [HIGH] `const_eval` rebuilds the whole const environment per call — 🔴 STILL PRESENT

- **Where:** `src/sim/value.rs:519–523`
- `ints.iter().map(|(k, v)| (k.clone(), *v)).collect()` clones every const/param name into a fresh
  `HashMap` on every call. Callers: `Index`, `Slice` (×2), `Replicate` in `eval` — i.e. every
  bit-select/slice, per settle, per cycle.
- **Fix:** make `checker::consteval::eval` take a borrowed map (its `Env` is just
  `HashMap<String, i128>`, `consteval.rs:24`).

### P2 [HIGH] `comb_env()` clones all simulation state per environment build — 🔴 STILL PRESENT

- **Where:** `src/sim/kernel.rs:252–255`
- `self.leaves.clone()` + cloned regs on every env build; built per `tick_edge`, `peek`, `eval`,
  `snapshot` — every `expect` in `mimz test` pays a full state clone and discards the comb memo.
- **Fix:** resolve `known` through two borrowed maps (`&self.leaves`, `&self.regs`) instead of
  merging into an owned one; optionally cache one env between state changes.

### P3 [HIGH] Per-frame String-keyed snapshots — 🔴 STILL PRESENT (Structurally Changed)

- **Where:** `src/sim/run.rs:34–38` (`Frame.values` is `BTreeMap<String, u128>`), `run.rs:199–212`, `kernel.rs:222–241` (`snapshot()` returns `Vec<(String, u128, Width)>` but still clones keys)
- `snapshot()` clones every signal name, then rebuilds a `BTreeMap<String, u128>` per frame —
  2 frames per cycle, all retained in `Timeline`.
- **Fix:** signal list is stable (comment at `run.rs:108`) — make `Frame.values` a `Vec<u128>`
  aligned with `Timeline.signals`. Kills the allocations and most frame memory at `MAX_SIM_CYCLES`.

### P4 [MED] `tick_edge` clones the full register map per edge — 🔴 STILL PRESENT

- **Where:** `src/sim/kernel.rs:183` — `self.regs.clone()` per cycle for hold-by-default.
- **Structural fix that subsumes P1–P3:** intern signal names to indices at elaboration and hold
  state in `Vec<Val>` — the per-edge copy becomes a memcpy of `Copy` Vals.

### P5 [MED] String allocation per memory read — 🔴 STILL PRESENT

- **Where:** `src/sim/kernel.rs:382` — `self.mem_cells.get(&(name.to_string(), addr))` allocates per
  `m[addr]` read. **Fix:** key mems as `BTreeMap<String, BTreeMap<u128, Val>>` so reads borrow `&str`.

### P6 [LOW] Parser clones every consumed token — 🔴 STILL PRESENT

- **Where:** `src/parser/mod.rs:136–141` — `bump()` (formerly `advance()`) clones; `Ident`/`Str`/`Int{raw}` carry Strings.
  **Fix:** `mem::replace` the slot with `TokKind::Eof` (tokens are consumed once).

### P7 [LOW] Identifier lexing allocates for keywords too — 🔴 STILL PRESENT

- **Where:** `src/lexer/mod.rs:295–310`; also `is_reserved` (`lexer/keywords.rs:99–101`) is a
  linear `Vec<String>` scan per non-keyword identifier.
  **Fix:** look up `&src[start..offset]` slice; allocate only for actual `Ident`; `HashSet` for reserved.
