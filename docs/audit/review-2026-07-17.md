## 1. Executive summary

Min-Mozhi is an unusually mature solo-built HDL compiler. The safety
engineering discipline — `#![forbid(unsafe_code)]`, release
`overflow-checks`, the "bound every count" doctrine, errors-as-values with 72
stable diagnostic codes, an honest append-only audit trail — exceeds most
commercial early-stage compilers. The language's core safety semantics
(lossless `+` vs. explicit `+%`, mandatory reset values, latch-free by
construction, `=`/`<-` split) are genuinely better than Verilog's defaults
and sit in the same design tier as Veryl/Spade.

Two things block production trust:

1. **One CRITICAL semantic defect, now proven:** the simulator and the
   emitted Verilog disagree on left-shift width semantics (BUG-11). Same
   source, same input: `mimz eval` returns 255, Icarus on the emitted
   Verilog returns 63. A user who trusts the simulator ships wrong hardware.
2. **One architectural root cause:** there is no single semantic authority.
   The checker's width model, the simulator's `Val` model, and the emitter's
   width strings each independently re-implement expression semantics, and
   the sim/eval/test paths **skip the checker entirely**. Every recurring bug
   class in the audit log (BUG-6, BUG-11, SEC-2, the S1/S2 panic class from
   the 2026-07-03 review) is this one defect wearing different clothes.

**Overall verdict: education-ready beta — yes. ASIC/FPGA signoff tool — not
until the top-5 items in §12 land.**

## 2. Language design — 8/10

### 2.1 Strengths (verified in spec/02 and the checker)

- **Lossless `+` (grows one bit) vs. explicit wrapping `+%`** — the dropped
  carry, the classic beginner bug, is a type error with a teaching message
  (spec/02 §1.1). Veryl and Spade arrived at the same idea; Min-Mozhi's
  diagnostics teach it better.
- **Mandatory `reg` reset value; a `reg` module must declare `reset`** — no
  uninitialized state is expressible.
- **Mandatory `else` on wire-driving `if`; exhaustive `match`** — accidental
  latches are inexpressible. This is the right constitution for both
  education and synthesis quality.
- **`=` (combinational) vs. `<-` (sequential, only inside `on rise/fall`)**
  with dedicated `clock`/`reset` declarations — eliminates Verilog's
  blocking/non-blocking trap wholesale.
- **Trilingual keywords with `translate` round-trip and localized
  diagnostics** — unique in the HDL space, built as real infrastructure
  (keywords.toml, messages.toml, Tamil inflection engine), not a veneer.
- Bundles with structural matching, payload enums, `foreach`, parametric
  widths, `clog2`, inline `test` blocks with `--emit-testbench` — a solid
  mid-tier feature set.

### 2.2 Findings

| ID  | Sev  | Finding                                                                                                                                                                                                                                                                                    |
| --- | ---- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| L1  | HIGH | **128-bit width ceiling** (BUG-13; simulator `Val` is `u128`). 256/512/1024-bit buses are routine (AXI, memory controllers, crypto). Hard blocker for real designs.                                                                                                                        |
| L2  | MED  | **No CDC construct yet.** Cross-clock-domain reads are a compile error (correct default — better than silent metastability), but the promised `sync` (spec, Phase 2) means multi-clock designs are currently unwritable.                                                                   |
| L3  | MED  | **Everything unrolls.** `repeat` folds to `REPEAT_BUDGET = 4096` iterations (`crates/mimz-core/src/lib.rs:11`); there is no true Verilog `generate` emission. Large parameterized structures (wide crossbars, big register files) hit the budget or bloat the netlist.                     |
| L4  | MED  | **Parser `MAX_DEPTH = 64`** (`crates/mimz-core/src/parser/mod.rs:111`). Correct DoS fix (SEC-1), and the E1113 message is good — but machine-generated sources (future IR round-trips, code generators) will hit it. Long-term: heap-based precedence-climbing removes the limit entirely. |
| L5  | LOW  | No `inout`/tri-state, no latch intent, no fixed-point (planned), no formal `prove` (planned). Fine for current scope; document explicitly as roadmap vs. non-goals.                                                                                                                        |

**Vs. peers:** safety semantics ≈ Veryl/Spade tier. Deliberately weaker than
Chisel/SpinalHDL/Clash on metaprogramming (host-language elaboration) —
a defensible trade for learnability, and the trilingual mission is unique.

## 3. Compiler architecture — 6.5/10

Pipeline: load → lex → parse → check (7 passes) → emit Verilog, with a
separate structural elaborator + event kernel for simulation. Errors are
values end-to-end with multi-error accumulation. The workspace split
(`mimz-core` pure pipeline / `mimz-sim` / `mimz-wasm` / fs-touching shell) is
textbook.

### 3.1 A1 [CRITICAL, root cause] — three semantic authorities, no IR

Expression semantics (widths, signedness, const folding) are independently
implemented three times:

- checker width model — `crates/mimz-core/src/checker/widths/`
- simulator value model — `crates/mimz-sim/src/sim/value.rs`
- emitter width rendering — `crates/mimz-core/src/emit_verilog/module.rs`

Recurring bug classes from this repo's own audit log that are all instances
of this one defect: BUG-6 (sim shift truncation), BUG-11 (sim shift
over-growth, §4.1), SEC-2 (sim const-eval was a divergent naive copy — fixed
by delegating to the checker's evaluator, which is exactly the right pattern
to generalize), and the S1/S2/S3 panic class from `review-2026-07-03.md`.
Fixing instances does not kill the class.

**Aggravating factor (A2, HIGH):** `mimz eval` / `sim` / `test` run
lex → parse → elaborate **without `checker::check`** (verified:
`src/commands/eval.rs` parses and evaluates directly; already called out as
S3 in the 2026-07-03 review and still true). The checker rejects `1 << 2` on
a bare literal (E0405, `ops.rs:493`); the simulator happily evaluates it
under different width rules. Two languages, one syntax.

**Recommendation (priority #1 and #2 in §12):** run the checker before every
sim path, then extract one shared width/const-eval module that checker and
simulator both call. The planned Phase-2 IR is the right long-term answer —
pull it forward and treat it as **correctness infrastructure, not a
synthesis feature**: emitter and elaborator should both consume elaborated
IR, never raw AST.

### 3.2 Other architectural notes

- **A3 [LOW] No incremental compilation, no parallelism.** Pure crates and
  per-file parsing mean a rayon par-iter over files is cheap to add later.
  Not urgent at current (benchmarked) speeds.
- Parser recovery is hardened with a progress guarantee (BUG-1 lesson);
  depth guards on all five recursion necks (SEC-1).

## 4. HDL correctness — 6/10

### 4.1 C1 [CRITICAL] — BUG-11 proven: simulation ≠ synthesis on `<<`

The checker/spec rule (spec/02 §3) says shifts **preserve the left operand's
width** — and the checker implements exactly that
(`checker/widths/ops.rs:487`, comment "width preserved (spec/02 section
3)"). The simulator instead **grows** the result width by the shift amount
(`mimz-sim/src/sim/value.rs:709-719`, `w = (l.width + shift).min(128)` — the
BUG-6 fix overcorrected). Emitted Verilog follows Verilog-2005
self-determined width, which matches the checker, not the simulator.

**Reproduced this review, end-to-end:**

```mimz
module Shifty {
  in a: bits[8]
  out y: bits[8]
  y = (a << 2) >> 2
}
```

- `mimz eval shifty.mimz --in a=255` → **`y = 255`** (high bits survive the
  `<<` in the widened intermediate, then shift back down)
- emitted Verilog `assign y = ((a << 2) >> 2);` under Icarus with `a = 255`
  → **`y = 63`** (the `<<` result is 8 bits; bits 8–9 are gone)

The program passes `mimz check` clean, so nothing warns the user. A design
validated in the simulator behaves differently in hardware — the worst
failure mode an HDL toolchain can have. `mimz test` shares the evaluator, so
a testbench can go green against the wrong semantics (the BUG-6 false-green
mechanism, now in the opposite direction).

**Fix:** the simulator must truncate the shift result to `l.width` (the
spec'd semantics), and a conformance test per operator must pin
checker/sim/Verilog agreement (§8). Note BUG-6's original complaint
(`1 << 2` on a bare literal) is checker-rejected as E0405 — the sim-side
width growth was never needed on checked input; it only "helps" on the
unchecked paths that A2 says should not exist.

### 4.2 C2 — BUG-12 corrected: consistent limitation, not divergence

`bugs.md` files BUG-12 as HIGH: "functions cannot access the enclosing
module's constants... breaking standard Verilog lexical scoping" at emit
time. **Verified this review: the checker rejects it too.** A module-const
reference inside a `fn` body fails `mimz check` with E0101 (`unknown name`)
— reproduced at HEAD. File-level consts in `fn` bodies work fine
(`examples/english/fn_with_const.mimz` demonstrates exactly this).

So checker and emitter **agree**: `fn` is a file-scoped construct that sees
file-level consts and its own params, and never module scope. That is a
**consistent language-design limitation** (module-parameterized helper
functions are inexpressible; workaround: pass the value as a `fn` parameter
or hoist the const to file level), not a checker/emitter divergence and not
broken scoping. **Recommend re-filing BUG-12 as a MEDIUM language-design
item** ("`fn` cannot be parameterized by module scope") with a spec ruling:
either bless file-scoping explicitly in spec/02, or design module-scoped
`fn` capture deliberately. The emitter's `file_env` swap
(`emit_verilog/module.rs:304`) is then correct as-is.

### 4.3 C3 [HIGH] — BUG-10 second half: bundle-typed `fn` returns emit invalid Verilog

Still open (params half fixed 2026-07-16). A bundle-typed `fn` return emits
syntactically invalid Verilog (`assign b_tx = as_uart(a)_tx;`) with **no
diagnostic**. Until the call-site-inlining feature lands, the emitter must
**reject with a diagnostic instead of emitting garbage** — silent invalid
output violates the project's own constitution (audit README threat model
item 3). Small fix; do it now.

### 4.4 Verified correct

- Signed values emit as Verilog `signed` vectors with the checker forbidding
  signed/unsigned mixing in one expression (E0403) — the soundness argument
  is documented in-code (`module.rs:1223-1228`) and holds.
- Sync reset stays clock-only; `async reset` correctly widens the
  sensitivity list to `posedge rst`.
- `rise`/`fall` → `posedge`/`negedge`; exhaustive `match` → full `case`
  coverage with the last arm as default.
- Icarus differential confirms listed examples byte-for-byte (§8 covers the
  list's coverage gap).

## 6. Performance — 7/10

- Measurement culture most compilers lack: Criterion per-phase benches,
  `mimz-bench` with committed `bench-history.jsonl`, memory profiling
  (memory-stats / dhat), CI perf batch.
- Current scale is fine (sub-second compiles on the example corpus).
- No incremental compilation, no parallel front-end — acceptable now; the
  pure-crate split leaves the door open (A3).
- **P1 [planning note]:** the BigUint migration for L1/BUG-13 will slow the
  simulator. Migrate behind a width threshold — `u128` fast path for ≤128-bit
  values, arbitrary-precision slow path above — and let the bench history
  catch regressions.

(`ALL_CHECKER_CODES`), `mimz explain` long-form explanations, localized
messages with the Tamil inflection engine, teaching hints (`+` error

guarantee. The one gap is R2: simulator runtime errors are stringly and
span-less compared to compile-time diagnostics.

## 9. Hardware-engineering practicality

- **Education / small FPGA / IoT: yes, today.** The hw-emulation
  peripherals (LED, speaker, TUI dashboard) are genuinely good teaching
  tools no competitor ships.
- **SoC / DSP / high-speed: not yet** — 128-bit cap (L1), no CDC (L2), no
  memory-inference control (BRAM vs. distributed), no pipelining aids, no
  timing-constraint story.
- **Safety-critical: no** — needs the conformance suite (T3) and formal
  `prove` first; the no-latch/no-uninit constitution is the right
  foundation to build that on.

## 11. Security & reliability — 8/10

The SEC-1..6 record shows real adversarial work: parser stack overflow,
OOM loop, DoS bounds everywhere, zero unsafe, import path traversal
inexpressible by construction (XID-only segments).

**Open: S5 [MEDIUM] — `[lib] std` override path traversal** (SEC-5 in
`security.md`). `src/config.rs` joins the `std` override path without
containment checks; a malicious `mimz.toml` can point the standard library
at arbitrary host directories (`std = "../../../../etc"`), which matters in
shared/CI/playground contexts. Fix: after canonicalizing, require the
resolved path to be inside the workspace root, or gate escapes behind an
explicit `--allow-external-std`. Close this week — it is small.

## 12. Top 10 priorities (ordered)

1. **C1 / BUG-11** — truncate the simulator's shift result to `l.width`
   (spec semantics); add per-operator checker/sim/Verilog conformance tests.
2. **A2** — run `checker::check` before every `eval`/`sim`/`test`/runner
   path. Kills the unchecked-AST family (S1/S2/S3, SEC-2 residue, C1
   reachability) at the root.
3. **C3 / BUG-10 returns** — diagnostic instead of invalid Verilog. Small
   and urgent.
4. **S5 / SEC-5** — sandbox the `[lib] std` override to the workspace root.
5. **C2 / BUG-12** — re-file as a language-design item; spec-rule `fn`
   scoping (bless file-scoping or design module capture deliberately).
6. **T2** — random-program differential fuzzing (mimz sim vs. Icarus).
   Highest-ROI test investment; structural defense for §3.1.
7. **A1** — one shared width/const-eval module for checker + sim; then the
   Phase-2 IR so emitter and elaborator consume elaborated IR, not raw AST.
8. **L1 / BUG-13** — arbitrary-precision `Val` behind a u128 fast path.
9. **L2** — ship the `sync` CDC construct; unblocks multi-clock designs the
   spec already promises.
10. **T1** — Icarus differential for all examples, CI-mandatory on every
    layer.

## 13. Roadmap recommendation

The phase order (IR → native backend → ecosystem) is right, with one
reframing: treat the IR as **correctness infrastructure, not a synthesis
feature**, and land it before more language surface. Every construct added
now — valid-bundle sugar included — is implemented three times (checker,
simulator, emitter); each one widens the divergence surface the IR must
later unify. Recommend one consolidation phase: freeze feature additions,
land items 1–7 above, then resume. The trilingual + safety-first identity
is strong enough that trust, not feature count, is what compounds.
